### 3. AI エージェントセキュリティ 2026 - プロンプトインジェクションが最大脅威
- **主要リスク**: LLM アプリケーションと AI エージェントに対するプロンプトインジェクションが首位
- **根本原因**: LLM がシステム命令とユーザー提供データを区別できない（単一トークンストリームとして処理）
- **「Lethal Trifecta」**: 以下の 3 つが揃うと致命的
  1. プライベートデータへのアクセス
  2. 信頼できない入力への曝露
  3. 情報漏洩ベクターの存在
- **攻撃手法の進化**:
  - **直接インジェクション**: 悪意あるコマンドを明示的に入力
  - **間接インジェクション**: メール、Web ページ、ドキュメント、画像に悪意ある命令を隠蔽
  - **マルチエージェント感染**: 相互接続されたエージェント間で悪意あるプロンプトが複製
  - **ハイブリッド攻撃**: XSS など従来サイバーセキュリティ手法と組み合わせ
  - **マルチモーダル攻撃**: 画像・音声に悪意ある命令を埋め込み
- **実例**:
  - **EchoLeak**: Microsoft 365 Copilot のゼロクリックプロンプトインジェクション
  - **CVE-2025-53773**: GitHub Copilot でリモートコード実行可能
  - **OpenClaw 監査**: 重要なプロンプトインジェクション脆弱性を発見
- **防御戦略**:
  - 最小権限原則、ランタイム制御、サンドボックス化
  - 人間-in-the-loop（重要・不可逆アクション）
  - 敵対的テスト・レッドチーミング
  - MAESTRO フレームワーク（エージェント AI 脅威モデリング）
- [Airia セキュリティガイド](https://airia.com/ai-security-in-2026-prompt-injection-the-lethal-trifecta-and-how-to-defend/)

```python
def validate_input(user_input: str) -> tuple[bool, str]:
    # 1. 悪意あるパターン検出
    if detect_injection_pattern(user_input):
        return False, "Suspicious pattern detected"
    
    # 2. 外部コンテンツの隔離
```

#### パターン 2: ランタイム監視
```python
class AgentGuardian:
    def __init__(self):
        self.action_whitelist = [...]
        self.sensitive_patterns = [...]
    
    def intercept_action(self, action: AgentAction):
        # 1. 許可リストチェック
        if action.type not in self.action_whitelist:
            raise SecurityException("Unauthorized action")
        
        # 2. 機密データアクセス検出
        if any(p in action.params for p in self.sensitive_patterns):
            require_human_approval(action)
```

#### パターン 3: 人間-in-the-loop
```
重要アクション分類:

├─ 確認必要: 書き込み・変更、中リスク
└─ 人間承認: 削除・外部送信・金融、高リスク
```
