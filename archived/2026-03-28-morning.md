## 🔍 HackerNews / General News

### 2026 年 3 月 - AI エージェント本番デプロイの課題

**「One-Model Trap」の警告:**
- 単一の大規模モデルですべてのタスクを処理しようとするのは非効率
- コスト増大、信頼性低下、複雑なタスクで不安定な動作を招く
- **解決策:** タスク特化型の小型モデル（SLM）を組み合わせる「constellation of narrow agents」

**本番環境の 5 大課題:**
1. **オーケストレーションの複雑化** - マルチエージェントシステムで急速に増大
2. **観測可能性** - 本番環境でのエージェント動作の監視・デバッグが困難
3. **コスト管理** - リソース集約的なエージェントシステムのスケーリング
4. **評価・テスト** - 一貫したベンチマークと成熟したツール不足
5. **ガバナンスと安全ガードレール** - 機能開発が先行しセキュリティが後手に

**注目フレームワーク（2026 年 3 月）:**
| フレームワーク | 用途 | 特徴 |
|---|---|---|
| **LangGraph** | 複雑な状態管理ワークフロー | 本番グレード、明示的制御、エラー処理 |
| **CrewAI** | ロールベースマルチエージェント | 高速プロトタイピング、直感的 |
| **Microsoft Agent Framework** | エンタープライズ統合 | AutoGen + Semantic Kernel 統合、.NET 対応 |
| **LlamaIndex** | データ集約型 RAG | ドキュメント中心、企業データ連携 |
| **Koog (JetBrains)** | Java エンタープライズ | Spring Boot 統合、フォールトトレランス |
| **Kaji (Shakudo)** | 自律型エージェント | プライベートクラウド/オンプレミス対応 |

## 🔗 MCP (Model Context Protocol) 新機能

### MCP Apps - インタラクティブ UI の時代

**2026 年 1 月導入:**
- **MCP Apps:** ツールがリッチな HTML インターフェースを返却可能に
- サンドボックス化された iframe 内で AI チャット体験に直接レンダリング
- ユーザーは対話インターフェースを離れずに操作可能

**対応プラットフォーム:**
- Claude, ChatGPT, VS Code など主要 AI プラットフォームでサポート
- OpenAI と共同開発

**ユースケース:**
- ダッシュボード操作
- デザイン編集
- フォーマット済みメッセージ作成
- ライブデータとの対話

**2026 年ロードマップ:**
1. **トランスポートの進化** - ステートレス実行、ロードバランサー対応
2. **スケーラブルなセッション管理** - サーバー再起動・スケールイベント透過化
3. **MCP Server Cards** - `.well-known` URL でサーバー機能メタデータを公開
4. **エンタープライズ対応:**
   - SSO 統合認証（Cross-App Access）
   - ゲートウェイ/プロキシパターン定義
   - 設定のポータビリティ
