## 🔍 HackerNews / General News

### OpenAI Agent SDK vs Google ADK - 公式 SDK 比較

**OpenAI Agent SDK (2025 年 3 月リリース):**
- 実験的 Swarm フレームワークから発展、本番グレードツールキット
- **コアプリミティブ:**
  - Agents: 指示とツールを持つ LLM
  - Handoffs: エージェント間タスク委任
  - Guardrails: 入出力検証
- Python ファースト（JS/TS 版もあり）
- 組み込みトレーシング（デバッグ・評価用）
- 100 以上 LLM 統合（OpenAI モデル最適化）
- HackerNews 2025-2026: 本番利用検討、柔軟性 vs 抽象化の議論

**Google ADK (Agent Development Kit):**
- Google オープンソース Python フレームワーク
- Vertex AI、Gemini モデル、Google Cloud サービスと密統合
- モデル非依存（他フレームワークとも互換）
- 2026 年初頭: ツール・インテグレーションエコシステム大幅拡張
- **特徴:**
  - グラフベースワークフロー
  - マルチエージェントアーキテクチャ
  - OpenTelemetry ネイティブ実行（観測可能性重視）
  - Model Context Protocol (MCP) サポート
- 2026 年ガイド：「開発者がエージェントアプリケーションを出荷可能に」

**比較:**
| 特徴 | OpenAI Agent SDK | Google ADK |
|---|---|---|
| 最適モデル | OpenAI | Gemini/Vertex AI |
| 統合 | 100+ LLM | Google Cloud + MCP |
| 観測可能性 | 組み込みトレーシング | OpenTelemetry ネイティブ |
| ワークフロー | ハンドオフベース | グラフベース |
| エコシステム | OpenAI 中心 | Google Cloud 中心 |

### n8n - AI エージェントワークフロー自動化プラットフォーム

**n8n の台頭:**
- オープンソース AI エージェントワークフロー自動化プラットフォーム
- ビジュアルワークフロー + AI 機能で複雑なマルチステップタスクを自動化
- 技術チームに推奨：拡張性、コスト効果、ソース利用可能

**インテリジェント自動化:**
- AI エージェントが「人間のように思考・決定・行動」
- 解釈、意思決定、生成、適応を必要とするタスクを処理

**実ワークフロー例:**
1. **AI チャットエージェント** - カスタマーサポート自動化、リアルタイム情報取得、会話コンテキスト維持
2. **ビジョンベース Web スクレイパー** - サイトレイアウト変更に適応、従来スクレイパーの限界克服
3. **AI データアナリスト / SQL クエリ可視化** - データ処理・分析の合理化
4. **メール要約 / 会議議事録** - 自動コンテンツ生成で生産性向上
5. **社内ドキュメントチャットボット / SIEM アラートエンリッチ** - 情報検索・セキュリティ運用改善
6. **GitHub API ヘルパー** - 開発タスク支援

**技術チームへのメリット:**
- オープンソース、実行ベース課金（ステップ課金と比較）
- SOC 2 コンプライアンス、シークレット管理、ロギング、デバッグ、RBAC
- 4,000+ スターターテンプレート（コミュニティ貢献）
- AI ワークフロービルダーで学習曲線緩和

**r/AI_Agents の評価:**
> 「n8n はコードレスとコードカスタマイズ（JS/Python）の両方に対応。技術ユーザーとエンタープライズチームに最適な選択肢」
