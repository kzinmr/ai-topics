# Claude Code Session Cache Cross-Account Leakage

July 2026 incident where Claude Code Enterprise ZDR workspace sessions experienced apparent cross-account cache leakage, with Sonnet 5 cache misses injecting unrelated content (Minecraft temple builder) from another account into a security-sensitive enterprise session. 313 HN points, 132 comments.

**Distinct from [[concepts/claude-code/claude-code-leak]]**: The March 2026 leak was an npm supply-chain attack exposing source code. This is a runtime session cache isolation failure affecting live enterprise users and their conversation data.

## Incident Details

### Bug Environment
- **Platform**: macOS (Apple_Terminal)
- **Claude Code version**: 2.1.199
- **Authentication**: Enterprise ZDR (Zero Data Retention) workspace
- **Model**: Sonnet 5 — every occurrence involved Sonnet 5
- **Trigger**: Cache miss after 5+ minutes of inactivity

### Observed Behavior

1. **Initial report** (milesrichardson-edb, 2026-07-04T02:04Z): Agent in an Enterprise ZDR session suddenly asked about "what kind of bricks for Minecraft temple" and asserted it was "building a Minecraft temple" in its recap — completely unrelated to the actual task (CSV data analysis for a procurement project).

2. **Local investigation**: `grep` of `~/.claude/projects/` found NO local trace of "temple" or "bricks" (the only match was `minecraft.py` from the Pygments lexer package, but no "temple" or "bricks" strings). This ruled out local context bleed.

3. **Cross-platform confirmation** (2026-07-04T13:53Z): Same user, **same Enterprise account, Claude Mobile session** — identical behavior: first response after 5+ minutes (cache miss) showed a "Detecting injection attempt" block followed by unrelated text. Common factor: **Sonnet 5, cache miss after 5+ minutes**.

4. **Second occurrence details**: The user asked for help with CSV file processing. Claude responded with "[...] I generated a mockup of what a 3-panel abstract print set in this space might look like above the sideboard..." with a "Detecting injection attempt, proceeding..." note in its response. This text was clearly from a different, design-oriented session.

### Severity Assessment

| Aspect | Assessment |
|--------|-----------|
| Cross-platform | Yes (Mac CLI + Mobile) — rules out local-only bug |
| Cross-instance | Appears to involve server-side cache serving wrong session content |
| Enterprise ZDR impact | If cache leakage crosses workspace boundaries, ZDR guarantees are violated |
| Bidirectional risk | If own session content can leak to other users, enterprise secrets are exposed |
| User-visible evidence | "Detecting injection attempt" suggests the model's safeguard fired but still served the leaked content |

## Technical Analysis

### Root Cause Hypothesis

The pattern (Sonnet 5, cache miss after 5+ minutes, cross-platform) suggests:

1. **Server-side prompt cache contamination**: Sonnet 5's cache layer may have a bug where cache keys overlap or cache entries are served to the wrong workspace/session after eviction and re-fill
2. **Compaction-related leak**: The text about "Detecting injection attempt, proceeding..." was generated by the model, indicating the model's own guardrail fired but the output still contained leaked content — perhaps a compaction/context window management error
3. **Enterprise isolation gap**: If cache is shared across workspaces in the same geographic region or cluster, ZDR workspace isolation is compromised

### Key Uncertainty

The reporter was logged into both a personal consumer account and an Enterprise ZDR workspace on the same machine — possibly using token switching. If the cache layer doesn't properly isolate by auth context, consumer session content could leak into Enterprise sessions. This is the most likely vector but also the least severe — the fix is to key cache by workspace ID.

The more severe scenario (cross-account leakage on the same workspace tier) is unconfirmed but cannot be ruled out.

Together, these three incidents represent a **complete security triad failure** for Claude Code in 2026: code (supply-chain), execution (sandbox), and data (cache).

## Implications for Enterprise AI Tooling

1. **Zero Data Retention guarantees are only as strong as cache isolation**: If cache can leak across workspaces, ZDR is meaningless for the cached window
2. **Sonnet 5-specific behavior**: The dependency on model version (only Sonnet 5 exhibited this) means model architecture changes can introduce new isolation vulnerabilities
3. **Monitoring gap**: The "Detecting injection attempt" guardrail fired but did NOT prevent the output — suggests detection without enforcement
4. **Enterprise adoption risk**: Combined with the sandbox escape (CVE-2026-55607), enterprise security teams face mounting concerns about Claude Code deployment
