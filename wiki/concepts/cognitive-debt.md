# Cognitive Debt

> **Note:** This page is a stub and needs expansion. See [[harness-engineering/agentic-workflows/cognitive-debt]] for the full content.

## Summary

Cognitive Debt is the concept describing the mental overhead that accumulates when:

- AI generates complex code without full human understanding
- Context windows become cluttered with generated code
- Future modifications become difficult due to lack of understanding
- The gap between what the code does and what humans understand grows

## Key Insights

From Simon Willison's cognitive debt concept:
- AI can generate code faster than humans can understand it
- This creates a growing "debt" of codebases that are hard to maintain
- Managing cognitive debt requires explicit effort in code legibility

## Relationship to Cognitive Load

Cognitive Debt is the AI-specific manifestation of cognitive load theory in software development.
