# synthetic-data

Synthetic data refers to artificially generated training examples created by AI models (typically LLMs) rather than collected from real-world sources. In the context of LLM pre-training, synthetic data augmentation has emerged as a critical solution to the **"data wall"** — where compute availability outpaces the supply of high-quality web data.

## Key Approaches

### Synthetic Rephrasing (Baseline)

The simplest approach: generate multiple synthetic versions of existing web documents using an LLM. Despite coming from a different distribution than real data, synthetic rephrases improve validation loss on real web data — a surprising finding confirming that diversity of expression aids generalization.

**Efficiency gain:** ~1.48× (at 32 rephrases per document)

### Megadocs (Structured Long-Form Synthetic Data)

Proposed by Kim et al. (2026, arXiv:2603.18534), **Megadocs** transform short synthetic rephrases into long, structured documents via two techniques:

| Technique | Method | Purpose |
|-----------|--------|---------|
| **Stitching** | Concatenate multiple synthetic rephrases of the same original into one long document | Expose model to long-range dependencies |
| **Stretching** | Insert rationales / additional context into a document | Increase depth and semantic richness |

**Efficiency gain:** ~1.80× (at 32 synthetic generations per original)

**Key finding:** The performance gap between Megadocs and simple rephrasing **widens** as synthetic data volume increases, suggesting better compute-loss scaling behavior. Megadocs specifically improve **long-context loss**, which simple rephrasing fails to address.

### Agentic Data Creation

A more advanced paradigm where AI agents actively curate, verify, and generate training data through multi-step processes. See [[concepts/autodata-agentic-data-creation]] for the full framework.

## Strategic Roles

Synthetic data serves two distinct purposes in the current AI landscape:

- **Scaling for Closed Models:** Companies like OpenAI and Anthropic use it to bypass the "data wall." Training models 100x larger than GPT-4 requires vast amounts of data that do not exist in human-written form.
- **Efficiency for Open Models:** Boutique providers use synthetic data (often generated by GPT-4) to fine-tune models like Llama or Mistral at a fraction of the cost of human labeling.

> **Key Insight:** The open and closed communities pursue very different end goals, yet synthetic data serves both.

## Constitutional AI as a Synthetic Data Framework

Anthropic's **Constitutional AI (CAI)** is the most sophisticated synthetic data pipeline in production, powering the Claude model family. CAI comprises two stages:

### 1. Principled Instruction Correction (Supervised Stage)
The model generates answers, critiques them against a written "constitution" (principles like "be helpful, honest, and harmless"), and revises them. The revised outputs become training data for supervised fine-tuning.

### 2. Principle-Following RLAIF (Reinforcement Stage)
The model generates pairwise preference data by evaluating which of two completions better follows a specific constitutional principle. This replaces human labelers in the RLHF process.

**Formula:** `CAI = Principled Instruction Correction + Principle-Following RLAIF`

CAI effectively creates **three types of synthetic data from one pipeline**: corrected instructions (SFT), preference pairs (RLHF), and critique signals — making it a unified synthetic data engine.

## Hierarchy of Synthetic Data Types

Difficulty and infrastructure requirements follow a clear progression:

| Level | Data Type | Use | Model Requirement | Open Example |
|-------|-----------|-----|-------------------|-------------|
| **1** | **Instructions** | Supervised Fine-Tuning (SFT) | Any capable LLM | Self-Instruct, Evol-Instruct |
| **2** | **Preferences** | RLHF / DPO training | Strong evaluator | UltraFeedback, Nectar |
| **3** | **Critiques** | Detailed feedback on prompt-generation pairs | 50B+ params | Prometheus (KAIST) |

Critiques are the "final frontier" — requiring high-capacity models to provide nuanced, detailed feedback that can meaningfully improve model outputs.

## Practical Tips

- **Always use the best model for generation:** Your model's quality is capped by its training data. Even if it requires paying for API access, the cost-benefit favors using the SOTA model (e.g., GPT-4) to generate training data.
- **Lock API Versions:** Model behavior in APIs can change (becoming more critical, returning nonsensical scores). Pin the specific model version for data consistency.
- **Robustness over Novelty:** Synthetic data helps models handle long-tail tasks and niche facts by allowing them to see obscure data points multiple times during training.

## Notable Open-Source Examples

### Models & Datasets
- **OpenHermes (Teknium):** High-performance models trained on synthetic instructions using a curated dataset of GPT-4 outputs.
- **Orca / Orca 2 (Microsoft):** Uses GPT-4 to augment and "clean" FLAN datasets for improved reasoning capabilities.
- **Starling (Berkeley):** RLHF model trained on the **Nectar** dataset — synthetic preferences generated by GPT-4.
- **Neural-chat (Intel):** Utilizes synthetic preferences for Direct Preference Optimization (DPO).

### Methods
- **Evol-Instruct:** A tree-of-rules approach to iteratively generate diverse, high-quality instructions by evolving seed instructions through complexity, constraint, and depth mutations.
- **SteerLM (Nvidia):** Uses attribute-based scores (helpfulness, humor, creativity) for fine-tuning without relying on OpenAI outputs — models learn to condition on desired attributes at inference time.
- **Prometheus (KAIST):** An open-source evaluator model designed specifically to provide AI feedback scores, enabling fully open-source evaluation pipelines.

## Superalignment and the Future of Synthetic Feedback

OpenAI's **Superalignment** team is tasked with using AI feedback to solve alignment for future superhuman models. The core assumption: as AI surpasses human capability, humans will no longer be able to provide sufficient feedback, making AI-to-AI supervision (synthetic feedback) the only path forward.

This vision places synthetic data at the center of the alignment agenda — not just as a training efficiency tool, but as the fundamental mechanism for steering models beyond human oversight capability.

## Pre-training Best Practices

- **Optimal Mixing:** The ratio of real to synthetic data must be carefully tuned; poor mixing can cause divergence
- **Epoching Strategy:** With optimal epoching, more synthetic generations improve accuracy without overfitting
- **Scaling Strategy:** When data-constrained, structure synthetic data into longer formats (Megadocs) rather than generating more short samples

## Distributional Shift Paradox

A counterintuitive finding: synthetic data helps improve validation loss on **real** web data, even though it follows a different distribution. The diversity of synthetic expressions appears to teach models more robust representations than repeated exposure to the same real documents.
