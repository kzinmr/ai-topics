# Agentic Manual Testing

**Exploratory testing** by coding agents. Leverages the agent's manual testing capability to discover issues (crashes, missing UI, edge cases) that automated tests miss even when passing.

## Core Principles

> "The defining characteristic of a coding agent is that it can execute the code that it writes."

> "Never assume that code generated by an LLM works until that code has been executed."

## Why Automated Tests Alone Are Insufficient

Even when automated tests pass, the following issues may be missed:
- **Crashes**: Runtime errors
- **Missing UI**: Display issues
- **Edge Cases**: Input patterns not covered by the test suite
- **Implementation Errors**: Tests pass but behavior contradicts intent

## Testing Mechanisms (By Code Type)

### Python Libraries
```bash
Try that new function on some edge cases using python -c
```

### Other Languages
```bash
Write code in /tmp to try edge cases of that function and then compile and run
```
→ Using `/tmp` avoids unintended commits to the repository

### JSON/Web APIs
```bash
Run a dev server and explore that new JSON API using curl
```
→ The keyword **"explore"** causes the agent to test multiple parameters/paths

## Browser Automation (Web UI)

### Main Tools
| Tool | Developer | Description |
|--------|--------|------|
| Playwright | Microsoft | Multi-language bindings |
| agent-browser | Vercel | Playwright CLI wrapper |
| Rodney | Simon Willison | Chrome DevTools Protocol-based |

### Why It"s Effective
> "Coding agents natively understand and leverage dedicated CLIs."

### Overcoming Historical Challenges
Traditionally "flaky" browser tests become more maintainable in the agent era:
- Agents automatically adapt to UI changes
- Developer friction for test updates significantly reduced

### Rodney Prompt Example
```bash
Start a dev server and then use `uvx rodney --help` to test the new homepage, look at screenshots to confirm the menu is in the right place
```

#### Engineering Techniques
1. **`uvx rodney --help`**: Automatic installation on first run + agent-optimized documentation
2. **Agent-First `--help`**: CLI help text structured for LLM consumption
3. **Vision Utilization**: `"look at screenshots"` explicitly triggers the agent"s visual verification capability
