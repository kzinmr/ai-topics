# Agentic Engineering

## 定義

Simon Willisonが提唱する、**コーディングエージェント（Claude Code、OpenAI Codexなど）を活用したソフトウェア開発手法**。2026年2月23日に[Agentic Engineering Patterns](https://simonwillison.net/guides/agentic-engineering-patterns/)として体系化を開始。

コーディングエージェントの定義的特徴：
> "They can both generate and execute code — allowing them to test that code and iterate on it independently of turn-by-turn guidance from their human supervisor."

## Vibe Coding との違い

| 次元 | Vibe Coding | Agentic Engineering |
|------|-------------|---------------------|
| ユーザー | 非プログラマー | プロフェッショナルソフトウェアエンジニア |
| 関与度 | コードに一切注目しない | 既存専門知識をエージェントで増幅 |
| 目的 | 動くコードを素早く作る | 品質と速度の両方を向上 |
| テスト | 省略されがち | 必須（Red/Green TDD） |

## Core Principles

### 1. Writing Code is Cheap Now
初期コード生成コストがほぼゼロになったため、**コードを書くことより、コードを管理・検証すること**に重点がシフト。

### 2. Hoard Things You Know How to Do
知識・技術の蓄積がエージェントにより指数関数的に価値を増す。

### 3. AI Should Help Us Produce Better Code
エージェントは速度だけでなく**品質向上**に貢献すべき。

### 4. Anti-patterns: Things to Avoid
**未検証コードを共同作業者に押し付ける**のが最大のアンチパターン。

## Key Patterns

### Red/Green TDD
テストファースト開発。エージェントが「動かないコード」や「不要なコード」を書くリスクを低減。
- **Red**: テストが失敗することを確認
- **Green**: テストが通るまで実装を反復

### First Run the Tests
4単語プロンプト`First run the tests`が持つ3つの効果：
1. テストスイートの存在をエージェントに認識させる
2. プロジェクトの複雑さの代理指標となる
3. エージェントを「テスト思考」に切り替える

### Agentic Manual Testing
> "Never assume that code generated by an LLM works until that code has been executed."

自動テストが通っても、クラッシュ・UI欠落・エッジケースは見逃される可能性がある。エージェントの探索的テスト能力を活用。

### Linear Walkthroughs
既存コードベースの理解を深めるため、エージェントに構造化されたコード解説を生成させる。

### Interactive Explanations
抽象的なアルゴリズムを、対話的アニメーションで直感的に理解可能にする。
