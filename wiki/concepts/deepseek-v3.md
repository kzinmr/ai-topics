# DeepSeek-V3

DeepSeek-V3は、**671B総パラメータ**（トークンあたり**37B活性化**）の Mixture-of-Experts (MoE) 言語モデル。DeepSeek-AI（中国）が2024年12月に発表。GPT-4oやClaude-3.5-Sonnetに匹敵する性能を、わずか **$5.576M（2.788M H800 GPU時間）** の訓練コストで達成し、オープンソースLLMのコストパラダイムを根本から変えたマイルストーン論文。

## アーキテクチャ

DeepSeek-V3は[[deepseek-v2|DeepSeek-V2]]のアーキテクチャを継承・発展させた設計。

### Multi-head Latent Attention (MLA)

KVキャッシュを低ランク結合圧縮（low-rank joint compression）で削減するアテンション機構。

```
c_KV_t = W_DKV · h_t          ← 圧縮潜在ベクトル（次元 d_c ≪ d_h · n_h · l_k）
k_C_t = W_UK · c_KV_t          ← アッププロジェクション（コンテンツ用）
k_R_t = RoPE(W_KR · h_t)       ← 分離されたRoPEキー（位置エンコーディング用）
```

- **推論時**: 圧縮潜在ベクトル `c_KV_t` と分離キー `k_R_t` のみをキャッシュ → KVキャッシュを大幅削減
- **効果**: V3ではMLAにより推論メモリ効率が劇的に改善。V2からの継承技術だが、V3のスケールで有効性を実証

### DeepSeekMoE

高効率なMoEアーキテクチャ。2つの主要特徴：

1. **細粒度エキスパート分割（Fine-Grained Expert Segmentation）**: 少数の大きなエキスパートではなく、多数の小さなエキスパートに分割 → より柔軟な知識の組み合わせ
2. **共有エキスパート分離（Shared Expert Isolation）**: 全トークンが常に通過する「共有エキスパート」を指定 → 共通知識の冗長な複製を防止

V3の構成：**1共有エキスパート + 256ルーテッドエキスパート**（トークンごとに上位8エキスパートを活性化）

### Auxiliary-Loss-Free Load Balancing（補助損失なし負荷分散）

従来のMoEモデルは、全エキスパートが均等に使われるように**補助損失（auxiliary loss）**を課していたが、これはモデル性能を劣化させる副作用があった。

DeepSeek-V3の革新的アプローチ：
- 各エキスパートに**動的バイアス項** `b_i` を追加
- **ルーティング時**: アフィニティスコア `s_i + b_i` でトップk選択（バイアス補正あり）
- **ゲーティング時**: 元のアフィニティスコア `s_i` で乗算（バイアス補正なし＝性能劣化ゼロ）
- **バイアス更新**: アンダーロードエキスパートの `b_i` を増加、オーバーロードエキスパートの `b_i` を減少

> 補助損失に頼らずに負荷分散を達成した最初の大規模MoEモデル。性能劣化ゼロでありながらエキスパート利用率の均等化に成功。

### Multi-Token Prediction (MTP)

各トークン位置で**複数の未来トークンを同時予測**する訓練目標。

- **構造**: メインTransformerの隠れ状態 `h_t` からD個の連続追加トークン `t+1, t+2, ..., t+D` を予測するMTPモジュールを追加（D=1で実装）
- **効果**: 訓練信号密度が上がり、データ効率が向上（14.8Tトークンの事前学習で優れた性能を達成した要因の一つ）
- **推論時活用**: MTPモジュールを**投機的デコーディング（speculative decoding）**に転用 → **1.8xの生成速度向上**（TPS: tokens per second）

## 訓練

### FP8混合精度訓練

671Bスケールで**FP8訓練の有効性を初めて検証**したマイルストーン成果。

| 量子化対象 | 方式 | 粒度 |
|-----------|------|------|
| 活性化（Activations） | タイル単位スケーリング | 1×128 |
| 重み（Weights） | ブロック単位スケーリング | 128×128 |

- **高精度累積への昇格**: Tensor CoreのFP8 GEMM内で128要素間隔でFP32累積に昇格し、精度低下を防止
- **メモリ効率**: FP16/BF16比で約50%のメモリ削減、通信量も半減

### DualPipeアルゴリズム

計算と通信を完全にオーバーラップさせる新規パイプライン並列戦略。

- **前方/後方パス**と**クロスノードall-to-all通信**を重ね合わせ → 通信オーバーヘッドをほぼゼロに
- **メモリ節約**: RMSNormとMLAアッププロジェクションを再計算（アクティベーション保存不要）、EMAパラメータをCPUメモリに配置
- **15日間の綿密な計画**: Forward/BackwardのCUDAカーネル配置を細かくスケジュールし、SMリソースの競合を回避

### 訓練コスト内訳

| 段階 | H800 GPU時間 | 推定コスト ($2/hr) |
|------|-------------|-------------------|
| 事前学習 | 2,664K | $5.328M |
| コンテキスト拡張（→128K） | 119K | $0.238M |
| ポストトレーニング | 5K | $0.01M |
| **合計** | **2,788K** | **$5.576M** |

### 訓練の安定性

全訓練過程で **回復不能な損失スパイクがゼロ**、ロールバックも不要。この規模のモデルでは極めて異例の安定性であり、アーキテクチャとシステム設計の堅牢さを示す。

### データセット

- 14.8兆トークンの高品質多様データ（Web、コード、数学、多言語）
- 文書パッキングとFill-in-Middle（FIM）戦略を採用
- トークナイザー：Byte-level BPE、語彙サイズ128K

## ポストトレーニング

### DeepSeek-R1からの蒸留

推論能力（検証・内省パターン）を[[deepseek-r1|DeepSeek-R1]]シリーズからV3に蒸留。これにより、ベースモデルでありながら強力なreasoning能力を獲得。

### GRPO（Group Relative Policy Optimization）

RLHFの代替として使用された強化学習アルゴリズム：
- ポリシーモデルと同サイズの**クリティックモデルが不要** → メモリ・計算資源を大幅節約
- グループ内の相対的報酬比較で方策を最適化
- 報酬モデル：Rule-based（数学・コーディング）+ Model-based（一般タスク）

### 長文コンテキスト

YaRNを使用して4K → 32K → 128Kトークンに拡張。2段階の段階的拡張により、長文タスクでの性能劣化を最小化。

## 歴史的意義

DeepSeek-V3は、以下の複数の理由からLLM発展史におけるマイルストーン論文と位置づけられる：

1. **コスト破壊**: GPT-4クラスの性能を$5.576Mで達成（Western frontier modelsの訓練コストは$100M+と推定される） → 「LLM訓練は巨額資本が必要」という前提を覆した
2. **FP8訓練の大規模検証**: 671BパラメータでのFP8訓練の実用性を初めて実証 → 以降のモデル開発の標準プラクティスに
3. **Auxiliary-Loss-Free Load Balancing**: MoEの長年の課題（補助損失による性能劣化）を解決 → 後続のMoEモデルに広く採用
4. **Multi-Token Prediction（MTP）**: 訓練密度向上 + 投機的デコーディングによる1.8x推論高速化 → 訓練・推論の両面で効率革命
5. **DualPipe**: 計算と通信の完全オーバーラップによる分散訓練の新標準
6. **オープンウェイト戦略**: MITライセンスでの完全公開 → グローバルな研究コミュニティの加速

この論文は、**「制約のある計算資源でも、アルゴリズムとシステムの革新によってフロンティア性能に到達できる」** ことを実証した。中国発のAI研究成果としても画期的であり、以降のDeepSeek V3.2、V4シリーズの基盤となった。

## ハードウェア設計への提言

論文は将来のAIハードウェア設計に向けて4つの具体的提言を行っている：

1. **通信オフロード用コプロセッサ**: all-to-all通信をSMから専用プロセッサに分離
2. **高精度FP8累積**: Tensor Core内でのFP8 GEMM累積精度の向上（CUDA Coreへの昇格不要に）
3. **細粒度量子化のネイティブサポート**: スケーリング係数をTensor Coreが直接処理
4. **オンライン量子化**: FP8キャストをメモリアクセス（TMA）と融合しHBM読み書きを削減
