# Monty Sandbox

**Monty** is a minimal, secure Python interpreter written in Rust, developed by Pydantic (Samuel Colvin). Designed for safely executing code generated by AI agents. Compared to traditional container-based sandboxes (Docker, E2B, etc.), it offers dramatically faster startup times in the **microsecond range**.

## Definition / Core Idea

Monty is built on the philosophy that "you probably don't need a full sandbox." Instead of spinning up an entire container, it provides a **custom Python runtime** that completely blocks access to the host environment. External interaction is only allowed through functions explicitly exposed by the developer.

## Architecture

```
LLM-generated Python code
        ↓
    Monty (Rust interpreter)
        ↓
   Ruff parser → bytecode compilation → execution
        ↓
   External function calls → Host Python/Rust/JS code
```

### Technical Features
- **Custom bytecode VM in Rust** — built from scratch, not a restricted version of CPython
- **Ruff parser** (Astral) to parse Python source
- **Built-in type checking** — ships `ty` (Ruff's type checker)
- **Snapshot-capable** — state can be serialized and restored on external function calls
- **Zero dependencies** — runs as a single Rust binary

## Performance

| Metric | Value |
|------|------|
| Startup (no type checking) | **4.5µs** (microseconds) |
| Startup (with type checking) | ~4.8ms |
| vs Container sandboxes | 100–10,000x faster |
| Runtime speed | 0.2–5x CPython (task-dependent) |

## Security Model

Monty uses an **allowlist** approach:
- **Filesystem**: Fully blocked by default
- **Environment variables**: Fully blocked by default
- **Network**: Fully blocked by default
- **External functions**: Only functions explicitly passed by the host can execute
- **$5,000 bug bounty** program (Hack Monty) active

### Capabilities

| Category | ✅ Supported | ❌ Not Supported |
|----------|-----------|-------------|
| Control flow | if/else, for, while, try/except | — |
| Data structures | dict, list, set, tuple, str, int, float | Class definitions (coming soon) |
| Functions | Function definition/call, lambda (limited) | — |
| Modules | Host-provided external functions only | import statement |
| Concurrency | async/await (partial) | Threads, processes |

## Use Cases

1. **AI agent code execution** — Safely run Python code generated by LLMs
2. **CodeMode** — Cloudflare's approach of "having LLMs write code instead of calling tools"
3. **Edge devices** — As a Rust binary, it can run on embedded systems and spacecraft
4. **In-browser execution** — Can compile to WebAssembly and serve as an in-browser sandbox (demonstrated by Simon Willison)

## Design Philosophy: The Runtime for Programmatic Tool Calling

Monty's README explicitly states that **Monty is a runtime designed for Programmatic Tool Calling**:

> *"Monty avoids the 'faff' of containers. It is designed for **Programmatic Tool Calling**, where LLMs write Python code to interact with tools more reliably than traditional JSON-based tool calling."*
> — Monty README

This means Monty is not merely a sandbox, but a **concrete runtime implementation** of the higher-level concept [[concepts/programmatic-tool-calling]].

### Trade-off Comparison (from Monty's docs)

| Technology | Security | Startup Latency | Snapshot | Setup |
|------|------------|--------------|----------------|------------|
| **Monty** | **Strict** | **0.06ms** | **Easy** | **Easy** |
| Docker | Good | 195ms | Intermediate | Intermediate |
| Pyodide | Poor | 2800ms | Hard | Intermediate |
| WASI/Wasmer | Strict | 66ms | Intermediate | Intermediate |
| YOLO Python | Non-existent | 0.1ms | Hard | None |

### Monty's Position in the 3-Layer Architecture

```
Programmatic Tool Calling (API mechanism) 
    └── Monty (Open Runtime — secure Python interpreter in Rust)
            └── CodeMode (Harness — wraps tools into run_code in PydanticAI)
```

- Monty sits in the **Open Runtime** layer, providing the execution foundation for the [[concepts/code-execution-with-mcp|Code Execution with MCP]] pattern
- [[concepts/pydantic-ai-harness]]'s CodeMode feature embeds Monty
- Its differentiator: achieving strict security while avoiding the "faff" of containers
