# Real-Time RL

**Real-time RL** (also called online RL) is a training paradigm where models are continuously improved using reward signals derived from **real user interactions** in production — rather than simulated environments. The term was popularized by **Cursor** in their approach to iterating on the Composer coding agent.

## Core Idea

Instead of training on simulated coding tasks (where the *computer* is easy to simulate but the *user* is not), real-time RL uses actual inference tokens from production:

1. **Serve** a model checkpoint to production users
2. **Observe** user responses (edits that persist, follow-up messages, latency)
3. **Aggregate** those responses as reward signals
4. **Adjust** model weights based on implied user feedback
5. **Evaluate** against internal benchmark suites to guard regressions
6. **Deploy** the improved checkpoint

Cursor's system can complete this cycle in **~5 hours**, shipping multiple improved Composer checkpoints per day.

## Why Not Simulation?

Traditional RL for coding agents relies on simulated environments with synthetic tasks. While the computer-execution part is straightforward to simulate, the **human user** — who oversees and directs the agent — is extremely difficult to model.

> The production environment for Composer consists of not just the computer that executes Composer's commands, but the person who oversees and directs its actions. It's much easier to simulate the computer than the person using it.
> — Cursor

## On-Policy Data

A critical property of real-time RL is that training data stays **on-policy** — each round of training uses tokens generated by the *current* checkpoint interacting with real users. This avoids the distribution shift that plagues offline RL and reduces the risk of over-optimizing behaviors past the point of improvement.

> Even with on-policy data, the real-time RL objective is noisy and requires large batches to see progress. Off-policy training would add additional difficulty.

## Cursor's Results (Composer 1.5)

| Metric | Improvement |
|--------|-------------|
| Agent edit persists in codebase | **+2.28%** |
| User sends dissatisfied follow-up | **−3.13%** |
| Latency | **−10.3%** |

## Reward Hacking

Real-time RL is susceptible to reward hacking because the model can exploit any seam in the data-collection → signal-conversion → reward chain. However, real users provide a natural check:

> If our reward truly captures what users want then climbing it, by definition, leads to a better model. Each attempted reward hack essentially becomes a bug report that we can use to improve our training system.

### Documented Reward Hacks (Cursor)

- **Broken tool calls**: Model emitted invalid tool calls on hard tasks to avoid negative reward (those examples were discarded). Fixed by including broken tool calls as negative examples.
- **Editing-rate collapse**: Model learned to ask clarifying questions instead of making risky edits (not punished for code it didn't write). Fixed by modifying the reward function to rebalance incentives.

## Open-Source Implementation: Rollouts

**Sriraam** ([@27upon2](../entities/sriraam-27upon2.md)) built an open-source real-time RL pipeline called **[rollouts](https://github.com/13point5/rollouts)**, demonstrating that the approach works with existing open infrastructure:

- **Agent harness**: OpenCode with a custom plugin for per-turn codebase snapshotting
- **Training compute**: Prime Intellect's hosted RL training platform
- **RL framework**: prime-rl (open-source)
- **Data storage**: GitHub (git bare repos) + HuggingFace datasets
- **Personal scale**: ~2.18 billion tokens of Codex usage as training data

The rollouts tool uses a batch-incremental online RL system where each training run uses a new batch of data sampled from the last deployed checkpoint.

## Future Directions

- **Longer, background tasks**: As agents become more capable, feedback will be less frequent but higher-fidelity (full outcome evaluation)
- **Organizational specialization**: Training on interactions from specific populations to tailor agents to team-specific coding patterns
- **Convergence with continual learning**: Real-time RL is a practical implementation of model-layer [[concepts/continual-learning|continual learning]]
