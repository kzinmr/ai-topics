# MRCR (Multi-Round Coreference Resolution)

## Overview

MRCR (Multi-Round Coreference Resolution) is a synthetic long-context evaluation benchmark that tests a language model's ability to distinguish between multiple pieces of target information ("needles") hidden within a large volume of superficially similar distractor text ("haystack"). Unlike the classic Needle-in-a-Haystack (NIAH) test where a single, obviously out-of-place needle is inserted, MRCR makes the needles blend in with the haystack by drawing both from the same distribution — all assistant responses are generated by the same underlying model (e.g., GPT-4o).

The benchmark was first introduced by Google DeepMind in their "Michelangelo" paper (Vodrahalli et al., 2024) as part of the Latent Structure Queries (LSQ) framework. OpenAI later released an open-source version (`openai/mrcr` on Hugging Face) that expanded the task's difficulty and provided reproducible data for the community.

## What It Measures

| Aspect | Detail |
|--------|--------|
| **Domain** | Long-context reasoning and retrieval |
| **Task type** | Multi-needle disambiguation and ordering in long synthetic conversations |
| **Format** | Text-only; multi-turn synthetic conversation → retrieve i-th instance of a repeated request |
| **Metric** | SequenceMatcher ratio (0–1) from Python's `difflib`, requiring correct alphanumeric hash prepended |

The model is given a long, multi-turn synthetically generated conversation between a user and an assistant. The user repeatedly asks for pieces of writing on specific topics (e.g., "write a poem about tapirs," "write a blog post about rocks"). Hidden in this conversation are 2, 4, or 8 identical ask instances. The model must return a specific one — e.g., "Return the 2nd poem about tapirs."

This task is challenging because:
1. **Distribution matching**: Needles are drawn from the same distribution as distractors — all assistant responses are model-generated, so nothing visibly "sticks out."
2. **Ordering reasoning**: The model must distinguish which instance (1st, 2nd, 3rd...) of a repeated request is being asked for.
3. **Scaling difficulty**: More needles (2 → 4 → 8) and longer contexts increase difficulty.

## Data Sourcing

| Detail | Value |
|--------|-------|
| **Total samples** | ~2,400 (100 per context-length bin) |
| **Needle configurations** | 2, 4, or 8 needles |
| **Context length bins** | (4K–8K], (8K–16K], (16K–32K], (32K–64K], (64K–128K], (128K–256K], (256K–512K], (512K–1M] |
| **Entity diversity** | 438 distinct entities |
| **Writing format diversity** | 10 distinct formats (poems, blog posts, social media posts, etc.) |
| **Generation model** | Needles and distractors both generated by GPT-4o |
| **License** | MIT (OpenAI version) |
| **Originating paper** | "Michelangelo: Long Context Evaluations Beyond Haystacks via Latent Structure Queries" (arXiv:2409.12640) |

All data is synthetically generated. The OpenAI version uses GPT-4o to produce all assistant responses, ensuring needle/haystack indistinguishability. A prepended alphanumeric hash is required in the answer; if missing, the match ratio is set to 0.

## Key Numbers

### Human Baseline
There is no formal human baseline for MRCR. The task is designed as a pure machine evaluation — humans would trivially score near-perfect on the disambiguation task, making it uninformative for human benchmarking.

### Top Model Scores (as of May 2026)

**MRCR v2, 2-needle @ 128K:**
| Model | Score |
|-------|-------|
| GPT-5 | 0.952 |
| GPT-4.1 | 0.572 |
| GPT-4.1 mini | 0.472 |
| GPT-4o | 0.319 |

**MRCR v2, 8-needle (most difficult variant):**
| Model | Score |
|-------|-------|
| Claude Opus 4.6 | 0.930 |
| GPT-5.5 | 0.740 |
| Gemini 3.1 Flash-Lite | 0.601 |
| Claude Sonnet 4.5 | 0.185 |

**Cross-context-length stability:**
- Gemini 3 Pro scores ~77% on 8-needle MRCR at 128K tokens but drops to just **26.3% at 1M tokens** — a ~50-point cliff.
- Claude Opus 4.6 scores 76% on the 8-needle 1M variant, demonstrating qualitative improvement in long-context reliability.

All frontier models in the original Michelangelo paper showed significant performance falloff before reaching 32K tokens, highlighting how difficult this task is even at moderate context lengths.

## @xeophon's Key Insight

> MRCR improves upon simple NIAH by placing multiple needles where the hay is made somewhat similar to the needles. The idea originated from Google; OpenAI's open-source version is likely to become the standard. This is an important long-context eval because it tests disambiguation — the model can't just spot something out-of-place; it must reason about which instance is being asked for among highly similar candidates.

## Strengths

1. **Harder than NIAH**: By making needles indistinguishable from distractors, MRCR eliminates the "needle visibly stands out" weakness of classic needle-in-haystack tests.
2. **Clean, automated scoring**: SequenceMatcher ratio + hash verification provides deterministic, unbiased evaluation without LLM-as-judge complications.
3. **Open-source and reproducible**: OpenAI's release on Hugging Face allows any researcher to replicate results.
4. **Fine-grained difficulty scaling**: 8 context-length bins × 3 needle configurations = 24 distinct difficulty levels, enabling detailed capability profiling.
5. **Tests ordering + retrieval simultaneously**: Requires both finding the right topic AND the right instance number — a realistic long-context reasoning scenario.

## Weaknesses

1. **Synthetic only**: The benchmark uses artificial conversations with no grounding in real-world long-context use cases (document QA, code repository understanding, book summarization).
2. **Narrow task scope**: Only tests one specific capability (disambiguation of repeated similar requests); doesn't cover synthesis, cross-document reasoning, or temporal understanding.
3. **No human baseline**: Cannot contextualize model performance against human ability.
4. **Single-generation model bias**: The OpenAI version uses GPT-4o for all needle/haystack generation, which may favor models with similar training distributions.
5. **Hash requirement is fragile**: Requiring a specific prepended string can penalize models that correctly retrieve the content but fail the formatting constraint.

## Comparison with Related Benchmarks

| Benchmark | Approach | MRCR Advantage |
|-----------|----------|----------------|
| **Classic NIAH** | Single needle, obviously different from haystack | MRCR uses indistinguishable needles; tests disambiguation, not just retrieval |
| **RULER** | Multi-needle but needles still identifiable | MRCR needles are from the same model distribution as haystack |
| **LongBench v2** | Real-world long-document tasks | MRCR provides clean, controlled difficulty scaling; complementary |
| **Graphwalks** | Graph traversal in long context | MRCR tests ordering + coreference specifically |

## The Michelangelo LSQ Framework

MRCR is one of three tasks in Google DeepMind's broader Latent Structure Queries (LSQ) framework, introduced in the "Michelangelo" paper. The framework's central metaphor: just as Michelangelo "chiseled away" marble to reveal sculpture, an LLM must "chisel away" irrelevant context to reveal latent structure.

The LSQ framework produces three evaluation types:
1. **MRCR** (natural language): Multi-round coreference resolution in synthetic conversations
2. **Latent List** (code): Finding a hidden list structure in large code files
3. **Multi-Round Co-reference** (code): Similar to MRCR but in code generation contexts

All three share the same design principle: the target information is hidden within structurally similar distractors, requiring the model to actively reason about context rather than passively retrieve.

## Practical Significance

MRCR has become a standard long-context benchmark for several reasons:

1. **GPT-4.1 release**: OpenAI prominently featured MRCR results in the GPT-4.1 announcement, establishing it as a key long-context metric.
2. **Claude Opus 4.6**: Anthropic highlighted the 8-needle 1M MRCR score (76% vs. Sonnet 4.5's 18.5%) as the headline metric for long-context improvement.
3. **Context Arena**: The community-run Context Arena website tracks MRCR across models and context lengths, making it one of the most transparently tracked long-context benchmarks.
4. **HELM integration**: Stanford's HELM framework includes OpenAI MRCR as one of its long-context evaluations, alongside RULER and ∞Bench.

The benchmark's ability to expose "context rot" — the degradation of model performance as conversations grow — has made it particularly valuable for enterprise deployments where long-context reliability is critical.
