# Self-Driving Codebases

A vision of autonomous software development where AI agents build, fix, and maintain codebases with minimal to no human intervention. Coined by [[entities/aman-sanger]] (CTO of [[entities/cursor-ai]]) as the "terminal state" of async coding agents.

## Core Components

### 1. Self-Healing and Fixing

Agents respond autonomously to events:

| Trigger | Agent Action | Human Role |
|---------|-------------|------------|
| Issue tracker | Agent proposes fix per issue | Review/merge or auto-merge |
| On-call page (2am) | Agent investigates, proposes solution | Single click to approve |
| Training run anomaly | Agent flags potential failures | Review flagged issues |
| Security vulnerability | Agent finds and patches PRs | Review security changes |
| Off-peak hours | Agent cleans up tech debt | No involvement |

The end state: agents as **primary on-call**, humans as **secondary escalation**.

### 2. Building Full Projects

Agents tackle entire products or extremely large features with little to no human intervention.

**Example**: Cursor built a working browser in a 1-week agent run:
- Billions of tokens consumed
- Tens of thousands of dollars in compute
- Rendering engine, JavaScript sandbox, animations
- Not yet production-quality, but functional

**Harness architecture** (recursive multi-agent):
```
High-level Planner
├── Sub-Planner A
│   ├── Worker 1
│   └── Worker 2
└── Sub-Planner B
    ├── Worker 3
    └── Sub-Planner C
        └── Worker 4
```

Each agent runs for a bounded duration, preventing the train-time/test-time mismatch that causes single agents to fail on very long tasks.

### 3. No Human-Written Code

The ultimate vision: all code in the codebase is generated by agents. Humans write **specs** — detailed, verifiable specifications that serve as both implementation plan and evaluation suite.

## Technical Foundations

### Train Time vs Test Time

RL-trained coding agents perform best when test tasks match their training distribution. A single agent running for tens of millions of tokens falls outside this distribution. Multi-agent systems solve this by keeping each subtask within the trained distribution.

### Model Specialization

Different models excel at different tasks:
- **Planning/orchestration**: OpenAI models
- **Computer use / multi-modal**: Gemini, Anthropic
- **UI creation**: Anthropic
- **Simple sub-tasks**: Faster, cheaper models (same quality for simpler work)

### Artifacts > Code Review

The paradigm shift in reviewing agent work:
- **Video artifacts**: Record the feature working, not the code implementing it
- **Research reports**: Summarize findings, not raw experiment code
- **Architecture diagrams**: Visualize design decisions
- **Disposable tests**: Verification artifacts that don't get checked in

## The Engineer's New Role

In a self-driving codebase world, engineers focus on:

1. **Spec writing**: Detailed, verifiable specifications (the new "prompt engineering")
2. **Product taste**: UX decisions, feature prioritization — hardest for models to learn
3. **Architecture**: Infrastructure and system design decisions
4. **Quality gates**: "Don't lose to slop" — maintaining standards against velocity pressure

Key insight: Engineers still need to "hold a lot in their head" — but at a higher abstraction level (entire codebase, not individual functions). Attention to detail shifts from code-level to product/architecture-level.

## Open Questions

- What is the right level of review for week-long agent runs? (Current best: artifacts at intermediate checkpoints)
- How to handle maximum ambiguity areas where the user didn't specify enough?
- How to make models better orchestrators of sub-agents? (RL training benefits workers more than orchestrators)
- What does "model UX" look like — producing useful, reviewable artifacts across all domains (not just product and research)?

## Current State (2026)

- Self-healing is already showing "serious signs of life" at Cursor (training monitoring, security review, issue automation)
- Browser project proves full-project building is possible but not yet shippable
- 30% of Cursor's merged PRs from cloud agents
- Spec-writing UX still requires many attempts and nudging
