# AI Organization — AI時代の組織論・ガバナンス・ワークデザイン

AIの自律実行が進む世界で、「人間の組織はどう変わるか」を体系的に整理するカテゴリ。
階層からインテリジェンスへ、管理から文脈提供へ、そしてソロ創業者/スーパーICの台頭まで。

## 階層構造

### Level 1: Fundamentals (基盤理論)
- **組織設計の歴史**: ローマ軍から現代のテック企業まで、ヒエラルキーがなぜ生まれ、なぜ崩れるか
- **情報ルーティング問題**: 階層の本質は「情報を適切な人に届けるメカニズム」
- **スパン・オブ・コントロール**: 人間が直接管理できる人数の限界 (3-8人)

### Level 2: Models (組織モデル)
- **[[ai-org-from-hierarchy-to-intelligence]]** — Block: 階層からインテリジェンスへの移行
- **[[ai-org-context-as-moat]]** — McKinsey/Reworked: Diamond型組織図とProprietary Context
- **[[ai-org-solo-founder-and-super-ic]]** — Reddit/FourWeekMBA: ソロ創業者とSuper ICの台頭

### Level 3: Synthesis (横断的テーマ)
- Hierarchy → Intelligence (報告ラインの削除、自律実行)
- Context as Moat (企業固有の文脈をAIに供給)
- Diamond Org Chart (組織図の逆転)
- Failure-Driven Roadmap (仮説駆動→現実駆動)
- Agentic Governance (リアルタイム監視とガードレール)
- MVP Commoditization (実行速度のコモディティ化)

## 3つの組織モデル

### 1. Block モデル (大企業向け)
Jack Dorseyが2024年に提唱。**World Model**と**Intelligence Layer**で中間管理を代替。
- 4つのビルディングブロック: Capabilities → World Model → Intelligence Layer → Interfaces
- 3つの新ロール: IC (専門技術者) → DRI (横断責任者) → Player-Coach (育成兼実務)
- **Failure Signal駆動**: Intelligence Layerが構成に失敗した箇所が未来のロードマップ
- モート: エコノミックグラフ (リアルタイム双方向トランザクションデータ)

> *"Most companies are focused on AI as a productivity enhancer. Few are focused on the potential of AI to change how we work together."*

### 2. McKinsey モデル (中堅〜大企業向け)
2025年9月提唱の**Agentic Organization**フレームワーク。
- 5つの柱: Business Model, Operating Model, Governance, Workforce, Technology
- **M-Shaped Supervisor**: 複数ドメインに跨り、AIチームをオーケストレーション
- **Diamond Org Chart**: 中央が最も厚い (AI + 人間エキスパート)、上下が尖った組織図
- **Agentic Budgeting**: AIによる予測・シナリオプランニング・リアルタイム監視
- AI完了可能タスク: 2019年以降7ヶ月ごとに倍増、2024年以降4ヶ月ごとに倍増

> *"AI飽和市場において、競合が真似できないのは自社固有の判断基準・文化・市場知見を構造化しエージェントに供給する能力。"*

### 3. Solo Founder / Super IC モデル (個人〜小規模向け)
Reddit r/ClaudeCodeとFourWeekMBAの議論から浮上。
- AIツールスタックで **1人 = 従来10人**
- 管理階層をバイパス → **「技術 → ビジネス → 戦略」の統合パス**
- **MVPのコモディティ化**: 実行速度はもはや差別化要因ではない
- 真のモート: **ドメイン知識 + 顧客理解**
- FrontPage時代の教訓: 「Shit in, shit out」 — 基礎スキルなしでは長期的に失敗

> *"If one person can build the product solo by using AI, so can everybody else. All it has really done is raise the bar for what is considered a minimum viable product."*

## 多次元比較

| 次元 | Block (Hierarchy→Intelligence) | McKinsey (Agentic Org) | Solo Founder/Super IC |
|------|------|------|------|
| **対象規模** | 大企業 (Block/Cash App) | 中堅〜大企業 | 個人〜小規模チーム |
| **コアアイデア** | World Model + failure signal | 5柱 + Diamond型 | AIツールスタック + ドメイン知識 |
| **意思決定** | 文脈駆動・自律実行 | M-Shaped Supervisor監督 | 即時・一人完結 |
| **AIの役割** | 協調メカニズムの再設計 | 専門エージェント (50-100/チーム) | 見えないチームメンバー (3-4人分) |
| **組織構造** | IC → DRI → Player-Coach | 機能サイロ廃止 → ワークチャート | 管理層なし |
| **モート** | エコノミックグラフ | Proprietary Context Layer | ドメイン知識 + スピード |
| **ロードマップ** | failure signal駆動 | 戦略計画 + Agentic Budgeting | 顧客フィードバック直結 |
| **最大のリスク** | 「壊れてから動く」移行期間 | ガバナンス能力がボトルネック | スケール時の技術的負債 |

## 横断的テーマ

| テーマ | 説明 | 関連 |
|--------|------|------|
| **Hierarchy → Intelligence** | 報告ラインの削除、意思決定権限の分散、透明性の最大化 | Block, McKinsey |

| **Diamond Org Chart** | 組織図の逆転。中央AI+エキスパート層が主力。エグゼクティブは監視・方向性設定 | Reworked |
| **Super IC / Solo Founder** | AIによる管理層のバイパス。技術的直接影響力の最大化 | Reddit, FourWeekMBA |
| **Failure-Driven Roadmap** | 仮説駆動PMから現実駆動（failure signal）への転換 | Block |
| **Agentic Governance** | エージェント実行に対するリアルタイム監視、ガードレール、エスカレーション | McKinsey, Agile Leadership Day |
| **MVP Commoditization** | 実行速度のコモディティ化。差別化はプロダクトビジョンとマネタイズへ | Reddit |
| **GIGO Principle** | 「Shit in, shit out」 — AI出力の品質は入力の品質で決まる | Reddit, FourWeekMBA |
