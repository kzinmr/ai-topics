# Zaimiri (@zaimiri)

**Zaimiri** is an AI productivity practitioner and content creator focused on personal AI operator workflows. Known for practical, non-hyped approaches to setting up and using AI assistants like Hermes Agent for real-world work.

## Overview

Zaimiri's core thesis is that **personal AI agents should be built slowly and deliberately**, layer by layer, rather than assembled in a chaotic weekend sprint. Their "7-day setup" methodology emphasizes:

1. **Day 1**: Verify the base agent works before customizing
2. **Day 2**: Define identity layer (tone, risk boundaries, when to push back)
3. **Day 3**: Add only high-signal, durable memory (preferences that matter in months)
4. **Day 4**: Move the agent into the interface you actually use (Telegram, etc.)
5. **Day 5**: Create skills from real repeated tasks, not theoretical workflows
6. **Day 6**: Add one quiet cron that only interrupts when there's signal
7. **Day 7**: Split profiles only when work needs genuine isolation (memory, permissions, audience)

## Core Philosophy: Personal Operator Over Chatbot

Zaimiri draws a sharp distinction between a chatbot and a personal operator:

> "A chatbot waits for prompts. A personal operator has context, procedures, tools, time, and boundaries."

The beginner mistake is trying to make the AI impressive immediately. The better move is to make it **reliable** — one clean memory, one clean skill, one useful Telegram lane, one quiet cron. Only then repeat and compound.

## Key Content Themes

### Memory Hygiene
Zaimiri advocates for **small, durable memory** that contains only facts that will still matter in a month:
- ✅ User preferences, stable conventions, common tools, repeated mistakes
- ❌ Temporary task progress, random links, one-day reminders, stale project status

> "The point of memory is not to archive your life. The point is to reduce repeated steering."

### Skill Design Philosophy
Skills should be procedural memory with clear boundaries:
- Trigger: a concrete, repeated task (not "be smarter")
- Workflow: specific files to inspect, commands to run, errors to watch for
- Output: defined delivery surface and receipt format
- Improvement path: learn from each execution

### Profile Isolation
Profiles should only be split when work needs different:
- Memory, identity, tools, permissions, credentials, audience, or delivery channel

> "A content agent and a family-office agent should not share memory. A client agent should not know your internal operating names. A coding agent should not carry your social-media voice."
