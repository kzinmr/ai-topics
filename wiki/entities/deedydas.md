## Core Ideas

### Meta Harnesses: Autoresearch on Steroids (2026)

Deedy coined the framing that popularized the meta-harness concept in the AI agent community:

> "Meta Harnesses is Autoresearch on steroids. Something I've been exploring recently is to get long running agents to hill climb on a verifiable task to continuously improve without my intervention."

**Key elements of his framing:**
- **Hill climbing**: Agents iteratively improve on verifiable tasks without human intervention
- **OS vs. Brain**: The harness (code around the model) is the optimization target, not the model weights
- **Karpathy lineage**: Builds on Autoresearch but takes it from exploration → optimization
- **Verifiable tasks**: The objective function is critical — you need a clear metric to climb

### Agent Design Patterns

Deedy has written extensively about coding agent design patterns, synthesizing lessons from systems like Devin, OpenHands, Aider, and Cursor:

| Pattern | Description |
|---------|------------|
| **Multi-agent** | Different roles with different tools and handoff capabilities |
| **Tool choice** | Which agent gets which tools; input/output design for each tool |
| **Underlying model** | Model selection for specific agent roles |
| **Diff format** | LLM-friendly diff formats for code editing |
| **LSP integration** | Language Server Protocol for static analysis and code retrieval |
| **Knowledge graphs** | Pre-processed cross-references searchable by keyword/fuzzy/embedding |
| **Fault localization** | SBFL (spectrum-based) and MBFL (mutation-based) techniques |
