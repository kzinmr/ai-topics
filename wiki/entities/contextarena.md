# Context Arena

**Context Arena** (contextarena.ai) is a live long-context benchmark leaderboard that evaluates 70+ large language models (LLMs) on their ability to retrieve and reason over information within extended context windows. It is built on the **GDM-MRCRv2** (Google DeepMind Multi-Round Coreference Resolution v2) benchmark, specifically the **8-Needle** variant — a rigorous multi-needle retrieval test that measures how well models can simultaneously track, distinguish, and recall multiple pieces of information buried within long synthetic conversations.

Unlike simpler benchmarks that test basic fact retrieval, Context Arena evaluates the full spectrum of long-context performance across **128K and 1M token** windows, tracking accuracy, AUC, token efficiency, and cost — providing both a capability ranking and a practical cost-effectiveness analysis for production deployments.

## Overview

Context Arena was built to address the gap between claimed context window sizes and actual usable capacity. Many models advertise large context windows but degrade significantly when tasked with retrieving information from the middle of long inputs — a phenomenon known as the "lost in the middle" problem. The leaderboard provides a standardized, reproducible way to compare models on this dimension.

The platform presents data across three difficulty tiers (2-needle, 4-needle, 8-needle), with the **8-Needle** variant being the hardest and most diagnostic. Results are organized into performance tiers:

| Tier | Models | 128k Accuracy | AUC @1M |
|------|--------|:------------:|:-------:|
| 🥇 Elite | GPT-5.5 (all variants) | 67.0% – 85.8% | 47.3% – 50.9% |
| 🥈 Strong | Claude Opus 4.6 / Sonnet 4.6 | 68.1% – 72.6% | 43.4% – 46.9% |
| 🥉 Mid-Tier | Gemini 3.1 Pro Preview / GPT-5.4 | 54.6% – 67.8% | 33.9% – 40.0% |
| ⚡ Budget/Free | DeepSeek V4 Pro/Flash, Mimo V2.5, Mistral Small-4 | 17.7% – 48.4% | 14.9% – 28.3% |

## Methodology

### GDM-MRCRv2 Benchmark

The GDM-MRCRv2 (Multi-Round Coreference Resolution v2) benchmark, introduced by Google DeepMind in the [Michelangelo paper](https://arxiv.org/abs/2409.12640), tests a model's ability to perform **multi-needle retrieval** across long synthetic conversations. The task structure:

1. A long, multi-turn synthetic conversation is constructed between a user and an assistant
2. The user sends multiple identical requests (e.g., "write a poem about tapirs") interleaved with distractor requests (e.g., "write a blog post about rocks")
3. The model is then prompted to return the *n-th* instance of a specific request (e.g., "Return the 2nd poem about tapirs")
4. All assistant responses are generated by the same model, so the needles blend naturally into the haystack — no obvious formatting differences tip off the model

### 8-Needle Test

The **8-Needle variant** is the hardest configuration, requiring the model to:
- Track **8 identical needles** hidden within the context
- Distinguish the **order** of those needles among numerous distractors
- Correctly identify and reproduce the *n-th* instance of a specific request

Context Arena offers three difficulty levels:
- **2 Needles** (Easy) — Basic multi-needle tracking
- **4 Needles** (Medium) — Moderate multi-needle complexity
- **8 Needles** (Hard) — Maximum difficulty, best differentiation between frontier models

### Context Windows

Models are evaluated at multiple token-length bins spanning two primary windows:

- **128K tokens (short-to-medium context)**: Tests performance up to ~128,000 input tokens, reflecting the context size most commonly available in production models
- **1M tokens (extended context)**: Tests performance at ~1,048,576 input tokens, pushing models to their maximum advertised context limits

Results are reported per bin, with the aggregated **AUC (Area Under Curve)** metric providing a single score that weights performance across all bins by bin width.

## Measured Metrics

Context Arena tracks five key metrics for each model:

### Accuracy
- **128k Accuracy (%)**: Correct retrieval rate at ~128K token context length
- **1M Accuracy (%)**: Correct retrieval rate at ~1M token context length
- Reported as raw percentage scores

### AUC (Area Under Curve)
- **AUC @128k**: Normalized area under the accuracy curve across all bins up to 128K tokens (scored 0–100%)
- **AUC @1M**: Normalized area under the accuracy curve across all bins up to 1M tokens (scored 0–100%)
- **Model AUC (%)**: AUC normalized only for context bins within the model's own max context length
- AUC is weighted by bin width, reflecting overall performance across the full range of context sizes

### Token Efficiency
- The ratio of input tokens consumed to output tokens produced, normalized relative to a baseline
- **Lower is better** — a value of 1.00x means standard token usage; values above 1.00x indicate higher token consumption for the same output
- Ranges from 0.95x (Grok-4.20) to 1.63x (Claude Opus 4.7)

### Cost
- **Input Cost ($)**: API cost for processing the input context
- **Output Cost ($)**: API cost for generating the output response
- Reported for both 128K and 1M context lengths
- Free models (DeepSeek V4 Pro/Flash, Mimo V2.5, Mistral Small-4) are marked with 🆓

### Badge System

| Badge | Meaning |
|-------|---------|
| 🏆 | Top 3 model for that metric |
| 🧠 | Reasoning was enabled for this variant |
| 🤝 | Run sponsored by a provider (API credits donated) |
| 🆓 | Free model or cost data unavailable |
| 🆕 | Recently updated results |
| N/A | No results for this model/bin (context window limit) |
| ⚠️ | Cost inaccuracy reported |
| 📉 | Incomplete cost data |
| 🗑️ | Model released >1 year ago |

### Key Performance Findings

- **GPT-5.5** dominates the leaderboard, maintaining >83% accuracy at 128K and ~50% AUC at 1M across all performance tiers
- **Claude Opus 4.6 & Sonnet 4.6** show consistent mid-to-high performance but degrade noticeably at 1M context
- **Gemini 3.1 Pro** exhibits sharp degradation at 1M context (drops to 25.9% accuracy), though competitive at 128K
- **Reasoning-enabled variants** (🧠 badge) consistently outperform their standard counterparts within the same model family
- **Free models** (DeepSeek V4, Mimo V2.5) provide respectable mid-tier performance at zero API cost
- **Context degradation** is prevalent — most models show significant accuracy drops between 128K and 1M

## Relationship to Other Long-Context Evaluations

### Needle-in-a-Haystack (NIAH)
The classic single-needle retrieval test popularized by Greg Kamradt's [LLMTest_NeedleInAHaystack](https://github.com/gkamradt/LLMTest_NeedleInAHaystack). Context Arena's GDM-MRCRv2 benchmark is a direct evolution:
- **NIAH**: Single needle, easily distinguishable from distractor text; tests basic retrieval
- **MRCRv2 (8-Needle)**: Multiple identical needles blended into the haystack; tests retrieval + ordering + coreference resolution
- The Michelangelo paper explicitly recommends MRCR as "a suitable default replacement" for vanilla NIAH, which they argue "fails to capture nuance in measuring model capabilities over long contexts"

### RULER
NVIDIA's [RULER](https://arxiv.org/abs/2404.06654) benchmark expands NIAH with 13 task types across four categories (retrieval, multi-hop tracing, aggregation, QA). Key differences from Context Arena:

| Dimension | Context Arena (GDM-MRCRv2) | RULER |
|-----------|---------------------------|-------|
| **Focus** | Multi-needle coreference resolution | 13 task types across 4 categories |
| **Difficulty** | 8-needle variant separates frontier models | Default settings allow high scores; harder variants not stress-tested |
| **Scope** | Narrower — single task type, deeper signal | Broader — multiple task dimensions |
| **Signal** | High signal even for smaller/pre-training models | Strong for retrieval tasks; less differentiation on harder variants |
| **Scoring** | Mean match ratio + AUC | Task-specific scores |

### LongBench v2
A realistic multi-task benchmark from Tsinghua with 503 MCQs across six categories, using real documents. Context Arena is complementary:
- **Context Arena**: Synthetic, controlled, high-signal diagnostic of retrieval ability
- **LongBench v2**: Real-world, multi-task assessment of deep understanding and reasoning
- Together they provide a more complete picture than either alone

### Michelangelo (LSQ Framework)
The [Michelangelo paper](https://arxiv.org/abs/2409.12640) (Kiran Vodrahalli et al., September 2024) introduced the **GDM-MRCRv2** benchmark that Context Arena uses, along with the broader **Latent Structure Queries (LSQ)** framework. The LSQ framework proposes moving beyond simple retrieval to evaluate models on identifying and reasoning over latent structures buried in noise — a philosophy Context Arena implements through its multi-needle coreference resolution task.

## Key Insights

1. **Context Window Size ≠ Usable Capacity**: Many models with large advertised context windows degrade significantly before reaching their limits
2. **8-Needle Separates Frontiers**: The hardest variant is the most diagnostic — GPT-5.5 variants maintain high scores while most models drop sharply
3. **Reasoning Boost**: Models with reasoning enabled (🧠) consistently outperform standard variants
4. **Cost-Performance Tradeoff**: Free models like DeepSeek V4 Pro provide respectable performance for budget-constrained deployments
5. **Token Efficiency Matters**: Prioritizing models with Tok Eff. ≤ 1.00x is recommended for cost-sensitive production use
6. **Context Degradation is the Norm**: Gemini 3.1 Pro and Grok-4.20 show particularly sharp accuracy drops at 1M context
