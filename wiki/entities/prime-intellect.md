**The Open Stack for Self-Improving Agents** — train, deploy, and continuously improve AI models on an integrated stack for compute, RL post-training, environments, evals, and inference.

## Products & Platform

### Lab
Post-training platform for building self-improving agents via [[reinforcement-learning]]. Auto-instruments coding agents (e.g., [[openai-codex-cli]], [[anthropic-claude-code]]) for RL fine-tuning. Includes hosted evaluations with 100+ open-source models and a public leaderboard. Key customers: Ramp, Zapier.

### Environment Hub
2,500+ open-source RL environments contributed by the community. Environments are the core primitive — they define tasks, verifiers, and reward signals for agent training.

Notable third-party environments: [[entities/sriraam-27upon2|Sriraam (@27upon2)]] built a [forkable OpenCode continual learning environment](https://app.primeintellect.ai/dashboard/environments/13point5/opencode-continual-learning) with per-turn codebase snapshotting for batch-incremental online RL — a Cursor-inspired real-time RL pipeline using Prime's hosted training.

### Compute
Decentralized GPU compute marketplace enabling distributed training across global resources. Supports pipeline-parallel decentralized inference.

## Autonomous Speedrunning Experiment (May 2026)

Prime Intellect tested **Codex (GPT-5.5)** and **Claude Code (Opus 4.7)** as autonomous research agents on the **modded-nanoGPT speedrun** — a competitive benchmark: train a 124M-param GPT-2 model to validation loss 3.28 in as few training steps as possible, using only optimizer/schedule/hyperparameter changes.

### Results

Across 3 waves (v1, novelty, v2, v3):
- ~10,400 runs, ~14,000 H200 GPU-hours consumed
- **Both agents surpassed the human baseline** (Muon at 3,500 steps) and the prior SoTA (3,225 steps)
- By v3: Claude Code reached **2,930 steps**, Codex reached **2,950 steps**
- The two agents pursued **independent strategies** — different optimizer combinations, schedules, and hyperparameters

### Agent Architecture

Each wave folder contains:
- `AGENTS.md` — Agent harness configuration and mission context
- `goal.md` — Specified goal and constraints
- `plan.md` — Agent's generated strategy
- `THREAD.md` — Chronological reasoning trail
- `runs.jsonl` — Structured run ledger with parsed metrics
- `variants/*.py` — Generated candidate training scripts

### Significance

This experiment demonstrates **LLM agents autonomously optimizing LLM training** — a capability milestone for AI research agents. The agents independently researched optimizer literature, proposed novel combinations, ran experiments, analyzed results, and iterated — all without human intervention in the optimization loop. The open-source release at [PrimeIntellect-ai/experiments-autonomous-speedrunning](https://github.com/PrimeIntellect-ai/experiments-autonomous-speedrunning) provides the complete experimental record.

See also: [[concepts/modded-nanogpt]], [[concepts/agentic-research]], [[entities/grad|Fares Obeid (Grad)]] — NanoGPT speedrun records.

## Design Philosophy

Prime Intellect's technical philosophy, articulated through the `renderers` library and RL stack:

1. **For RL, the inference server should be a simple token-only endpoint** — Token-In, Token-Out
2. **Environments should be oblivious to tokenizers** — usable as evals for arbitrary model APIs
3. **Every assumption about chat templates will be violated eventually** — plan for it
4. **Official chat templates are often "wrong"** — require explicit opt-in repair
