# Microsoft VibeVoice

VibeVoice is a novel framework from **Microsoft Research** for generating **expressive, long-form, multi-speaker** conversational audio (podcasts, dialogues) from text. It is a **next-token diffusion** model that combines a large language model (LLM) with continuous speech tokenizers operating at an ultra-low frame rate of **7.5 Hz**. The paper was accepted as an **ICLR 2026 Oral** and was released as open-source under the MIT license before being temporarily disabled due to misuse concerns.

## Architecture

VibeVoice employs a three-component architecture:

```
Text + Voice Prompt → LLM (Qwen2.5) → Diffusion Head → Acoustic Decoder → Speech
```

### 1. Continuous Speech Tokenizers

**Acoustic Tokenizer** (~340M params, encoder/decoder):
- Based on σ-VAE variant (from LatentLM, Sun et al. 2024)
- Mirror-symmetric encoder-decoder with 7 stages of modified Transformer blocks
- **3200× downsampling** from 24kHz input — 80× better compression than Encodec
- Operates at 7.5 Hz (2 speech tokens ≈ 1 BPE text token)

**Semantic Tokenizer:**
- Mirrors acoustic architecture (without VAE components)
- Trained with ASR proxy task to capture linguistic content

### 2. Language Model (LLM)

- **Base**: Qwen2.5-1.5B (1.5B model) / Qwen2.5-7B (7B model, reported but unreleased)
- **Context length**: 65,536 tokens (curriculum: 4K → 16K → 32K → 64K)
- Understands textual context and dialogue flow
- Generates autoregressive latent vectors

### 3. Diffusion Head

- Lightweight module: **4 layers, ~123M parameters** (1.5B model) / ~40M (0.5B streaming)
- Predicts acoustic VAE features using DDPM (Denoising Diffusion Probabilistic Models)
- Uses Classifier-Free Guidance (CFG) and DPM-Solver during inference
- Generates high-fidelity acoustic details from LLM hidden states

| **VibeVoice-Large** | — | — | — | ~45 min | — | Disabled |

### VibeVoice-Realtime-0.5B

A lightweight streaming variant:
- Single speaker only (no semantic tokenizer)
- Interleaved windowed design: encodes text chunks while generating audio from prior context
- ~300ms first audible latency (hardware dependent)
- Deployment-friendly at 0.5B parameters
- Supports streaming text input (ideal for LLM voice agents)

## Key Capabilities

| Capability | Detail |
|------------|--------|
| **Max duration** | Up to 90 minutes per generation |
| **Max speakers** | Up to 4 distinct speakers |
| **Languages** | English, Chinese (primary); some multilingual capability |
| **Expression** | Emotion, singing, background music, breaths, lip smacks |
| **Context window** | 65,536 tokens |
| **Frame rate** | 7.5 Hz (3200× compression) |
| **License** | MIT (open source) |

## Benchmark Performance

- **ICLR 2026 Oral** acceptance
- Outperforms **Gemini 2.5 Pro TTS** and **ElevenLabs V3** in subjective evaluations
- **WER**: 1.29% (VibeVoice-7B) vs 1.73% (Gemini)
- **Speaker similarity**: 0.692 (competitive with commercial systems)
- Evaluation: 24 human annotators × 6 hours each across 8 long conversational transcripts

## Data Pipeline

Microsoft developed a specialized annotation pipeline that generates:
- Pseudo transcriptions from raw podcast audio
- Turn-taking labels for multi-speaker segmentation
- Enables learning of authentic conversational dynamics (natural turn-taking, pacing, non-lexical cues)

## Safety & Availability

### Safety Features
- **Audible disclaimer**: "This segment was generated by AI" embedded in every audio file
- **Imperceptible watermarking**: For provenance verification
- **Logging**: Inference requests are hashed and logged for abuse detection

### Repository Status
> **2025-09-05**: Microsoft disabled the GitHub repository after discovering the tool was used in ways "inconsistent with the stated intent." The repo remains disabled as of the time of writing, though Hugging Face model weights and the technical report remain available.

### Out-of-Scope Uses
- Voice impersonation without explicit, recorded consent
- Real-time "live deep-fake" applications
- Background ambience, Foley, or music generation
- Languages other than English and Chinese

## Limitations
- No overlapping speech modeling (speakers don't talk over one another)
- May inherit biases from Qwen2.5 base model
- Research-only recommendation (not for commercial use without further testing)
- Multi-lingual quality is unpredictable for non-primary languages
