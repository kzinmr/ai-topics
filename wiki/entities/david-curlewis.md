## Key Writings

### "Lines of Code Got a Better Publicist" (June 10, 2026)

Curlewis's most prominent essay argues that AI vendors have shifted from falsifiable outcome claims (e.g., GitHub Copilot "55% faster") to unfalsifiable volume metrics (e.g., "75% of code is AI-generated"). The industry spent decades learning that lines of code and PR counts are poor productivity measures — AI vendors have simply repackaged the same flawed metric with better marketing.

Core arguments:

- **Volume ≠ Outcome**: "75% of code is AI-written" can be true and keep increasing regardless of whether anything actually improved — faster delivery, fewer incidents, happier customers.
- **Research contradiction**: Anthropic's own RCT showed AI-assisted developers scored 17% lower on code comprehension with no statistically significant productivity gain — while the same company claims "8x more code shipped."
- **Real-world impact**: Block cut 40% of workforce citing AI, Atlassian cut 10% — volume metrics are moving budgets and headcount plans, not just PR slides.
- **The litmus test**: *"Is that an outcome, or a volume?"* Applied to any AI vendor claim, this question quickly deflates positions built on vanity metrics.

### "Calibrating on Capability, Not Activity" (May 2026)

Earlier essay exploring the distinction between what AI tools *can* do and what they *do* do in practice — emphasizing the gap between demos and real workflows.

### "My AI Context System" Series (Feb–Mar 2026)

Multi-part series on building personal AI context management systems — practical exploration of how developers integrate AI tools into daily work, with an emphasis on measuring what actually changes rather than what tools are used.

## Core Perspective

Curlewis's work represents a **metrics-first critique of AI adoption narratives**. He doesn't argue that AI tools provide no value — he argues that the metrics used to claim value are systematically broken, and that the industry already knows how to measure engineering outcomes (DORA metrics, reliability, rate of meaningful change). His challenge: if AI really made everyone more productive, why aren't companies using that freed capacity to deliver more customer value rather than cutting headcount?
