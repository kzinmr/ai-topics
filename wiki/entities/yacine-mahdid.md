## Overview

Yacine Mahdid is a software engineer and computational neuroscience researcher based in Ottawa, Canada. He works at a Canadian startup and has previously been affiliated with McGill University (Computational Neuroscience) and Biapt Lab. His work spans practical tooling for AI-assisted development, creative computing, and hardware design.

Mahdid is best known as the creator of dingllm.nvim (865+ GitHub stars), a lightweight Neovim configuration for LLM-assisted programming. The project embodies his philosophy of minimal, hackable tools that get out of the user's way. His other notable projects include talk (588 stars — a TypeScript project for making sand talk), dingcad (473 stars — live reload CAD scripting because "lmao UIs"), scribepod (174 stars — automated AI podcast generation), and just-large-models (44 stars — minimal LLM implementations with "as little abstraction as possible").

Mahdid's GitHub bio reflects his approach: "free yourself, brothers and sisters." His dingllm.nvim documentation famously reads: "read the code dummy" — a statement of radical transparency and self-reliance. He believes that users should understand the tools they use, not treat them as black boxes.

His work is characterized by a blend of practical engineering and philosophical inquiry. The talk project explores the intersection of code and art (generating visual patterns from code). dingcad challenges the conventional wisdom that CAD tools need complex UIs, instead advocating for a code-first approach to hardware design. scribepod automates podcast generation using AI, demonstrating how modern tooling can replace labor-intensive production workflows.

Mahdid runs a Substack newsletter, "learning with yacine," described as "untangling the complexity of deep learning and the brain — ideal for scholars, industry practitioners or toddlers." The newsletter has over 1,000 subscribers and reflects his commitment to making complex technical topics accessible.

He is active on X (@yacineMTB) where he shares technical insights, advocates for open-source tooling, and engages with the broader AI/ML community. His personal site at yacine.ca features a video (practice.mp4) that embodies his philosophy of learning through doing.

## Core Ideas

### Radical Minimalism in Tooling

Mahdid's projects consistently reject complexity in favor of transparency. His dingllm.nvim documentation says: "This is a really light config. I will be pushing breaking changes. I recommend reading the code and copying it over." This is not a bug — it's a feature. He wants users to understand every line of code they run.

The just-large-models repository makes this explicit: "Just large language models. Hackable, with as little abstraction as possible. Done for my own purposes, feel free to rip." The goal is not to build a polished product but to create a working reference that others can learn from and modify.

### Code as Medium

The talk project (588 stars) exemplifies Mahdid's view of code as a creative medium. By generating visual sand patterns from code, he demonstrates that programming is not just about building functional software — it's about creating experiences, exploring aesthetics, and expressing ideas.

This philosophy extends to dingcad, which challenges the assumption that CAD tools need graphical interfaces. By making hardware design scriptable and live-reloading, Mahdid treats CAD as code — something that can be versioned, diffed, and composed.

### Learning by Doing

Mahdid's approach to education is deeply practical. His "learning with yacine" Substack newsletter aims to make complex topics accessible to everyone — "scholars, industry practitioners or toddlers." The helps repository contains code he's written to teach others, demonstrating his commitment to knowledge sharing.

His GitHub bio — "free yourself, brothers and sisters" — captures this ethos. The goal is not to create dependency on tools but to empower users to understand, modify, and build their own solutions.

### AI-Assisted Development as Extension of Self

dingllm.nvim and llm.nvim are not just tools — they're extensions of the developer's workflow. By integrating LLMs directly into Neovim (the editor of choice for many serious developers), Mahdid makes AI assistance feel like a natural part of the programming process rather than a separate, foreign application.

The system prompts in dingllm.nvim are carefully crafted: "You should replace the code that you are sent, only following the comments. Do not talk at all. Only output code." This is a pragmatic approach — the AI is a tool, not a conversational partner. It should do its job and get out of the way.

### Automated Media Production

scribepod (174 stars) demonstrates Mahdid's interest in automating content creation. The project generates AI podcasts from scripts, replacing the labor-intensive process of recording, editing, and producing audio content with code.

This is part of a broader trend: using AI to democratize media production. If a single person with a script and some Python can produce a podcast that sounds professional, the barrier to entry drops dramatically.

### Open-Source Hardware Design

Mahdid's recent contributions to tscircuit/tscircuit-autorouter (open-source PCB routing) and his dingcad project reflect an interest in democratizing hardware design. By making CAD tools scriptable and open-source, he enables a broader community to participate in hardware development — not just those who can afford expensive proprietary tools.
