# Aman Sanger

Co-founder and CTO of [[entities/cursor-ai]], the AI-native code editor. Together with co-founders, built Cursor into one of the most widely used AI coding tools, with the majority of code written in Cursor now coming from agents rather than manual tab completions.

## Key Contributions

- Led Cursor's transition from autocomplete (Era 1) to synchronous agents (Era 2) to async cloud agents (Era 3)
- Pioneered the "artifacts" paradigm — reviewing agent-produced videos, research reports, and architecture diagrams instead of raw code diffs
- Introduced the concept of "self-driving codebases" — autonomous code maintenance with self-healing, auto-fixing, and full-project building
- Championed multi-agent architecture with model specialization (OpenAI for planning, Anthropic/Gemini for computer use and UI)

## Vision

Sanger sees the future of engineering as writing detailed, verifiable specs rather than code directly. Engineers will hold entire codebases at a higher abstraction level, focusing on taste, architecture, and product decisions. His key warning: "Don't lose to slop" — maintain quality standards even as AI velocity increases.

## Speaking

- **"Building Towards Self-Driving Codebases with Long-Running, Asynchronous Agents"** (Apr 2026): 37-minute GTC talk — outlined three eras of AI coding, demonstrated Cursor Cloud Agent, introduced self-driving codebase vision. ## Notable Points

- Cursor is scaling engineering 3x in 2026 despite AI productivity gains — believes in compounding excellent engineers with excellent tools
- Internal metric: 30% of merged PRs from cloud agents (as of Feb 2026)
- Cursor's interview process: 1-2 day onsite project using agents, scope increased as agents improved
