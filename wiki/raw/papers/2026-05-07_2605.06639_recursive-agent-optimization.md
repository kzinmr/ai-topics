# Recursive Agent Optimization

**Authors**: Apurva Gandhi¹, Satyaki Chakraborty², Xiangjun Wang², Aviral Kumar¹, Graham Neubig¹
**Affiliations**: ¹Carnegie Mellon University, ²Amazon AGI Labs
**arXiv**: [2605.06639](https://arxiv.org/abs/2605.06639)
**Project Page**: https://apga.github.io/RAO

## Abstract

We introduce **Recursive Agent Optimization (RAO)**, a reinforcement learning approach for training recursive agents: agents that can spawn and delegate sub-tasks to new instantiations of themselves recursively. Recursive agents implement an inference-time scaling algorithm that naturally allows agents to scale to longer contexts and generalize to more difficult problems via divide-and-conquer. RAO provides a method to train models to best take advantage of such recursive inference, teaching agents **when** and **how** to delegate and communicate.

Key findings:
- Better training efficiency compared to single-agent systems
- Can scale to tasks that go beyond the model's context window
- Generalize to tasks much harder than training tasks
- Reduced wall-clock time compared to single-agent systems

## Core Concepts

### Recursive Agents
Agents that can spawn and delegate sub-tasks to new instantiations of themselves recursively. Each sub-agent gets a fresh context window, expanding effective working memory. The execution produces a dynamically structured recursive **execution tree** where each node corresponds to one agent instance.

### RAO Training
- **Single policy** is instantiated at every node of the execution tree
- The same model learns both task-solving AND delegation/subtask-generation
- Supports **dynamically generated** hierarchies (not fixed/designed)
- Training objective can be thought of as multi-task with tasks sampled from different tree levels
- Dense credit assignment per sub-task trajectory via environment feedback or LLM-judge proxy

### Relation to Recursive Language Models (RLMs)
The harness implementation resembles RLMs by Zhang et al. (2025), with two key differences:
1. Demonstrates positive results for recursion depths > 1
2. Supports **concurrent** sub-agent execution via async delegation primitive

### Key Results
- Training Qwen3-4B-Instruct for only **75 steps** on deep research → recursive agents beat trained single agent baseline by **16% success rate** (SR)
- Recursive agents adapt inference-time compute based on problem difficulty → max recursive depth increases as problems get harder
- Better training efficiency from structured, dense credit assignment
- Generalizes to tasks harder than training distribution

## Relation to Sub-Agent Systems
This work directly addresses sub-agent systems like Claude Code, Codex CLI, and OpenCode that use sub-agents as an inference-time scaffolding. RAO trains the model itself to use these scaffolds effectively, rather than treating them as external wrappers.
