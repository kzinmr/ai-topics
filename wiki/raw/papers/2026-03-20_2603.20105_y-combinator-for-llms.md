## Abstract

The paper introduces **λ-RLM**, a framework that replaces the open-ended REPL-based control flow of standard Recursive Language Models (RLMs) with a **typed functional runtime grounded in λ-calculus**. The core insight is framing recursive reasoning as a fixed-point over a library of pre-verified combinators (SPLIT, MAP, FILTER, REDUCE, CROSS), with neural inference restricted to bounded leaf subproblems.

## Key Novelty Claims

1. **Typed functional runtime** — replaces LLM-generated control code with deterministic, pre-verified combinators
2. **Formal guarantees** — termination proof, closed-form cost bounds, optimal partition theorem (k*=2), power-law accuracy scaling vs exponential decay
3. **Scale substitution** — 8B model + λ-RLM matches 70B model + standard RLM

## Methodology

The framework is defined by the lambda term:
λ-RLM ≡ fix(λf.λP.if |P| ≤ τ* then M(P) else REDUCE(⊕, MAP(λp_i. f p_i, SPLIT(P, k*))))

Where:
- M = base LLM (neural oracle for leaf subproblems)
- k* = optimal partition size (proven: k*=2 for standard cost models)
- τ* = base-case threshold (max sub-prompt length)
- ⊕ = task-dependent composition operator

### 5-Phase Execution Pipeline
1. REPL Initialization — store prompt P
2. Task Detection — single LLM call to select task type
3. Planning — deterministic calculation of k*, τ*, depth d
4. Cost Estimation — bound number of oracle calls pre-execution
5. Execution — run pre-built combinator chain

### Combinator Library
| Combinator | Type Signature | Description |
|-----------|---------------|-------------|
| SPLIT | Σ* × N → [Σ*] | Partition string into k chunks |
| MAP | (α→β) × [α] → [β] | Apply function to every element |
| FILTER | (α→B) × [α] → [α] | Retain elements satisfying predicate |
| REDUCE | (β×β→β) × [β] → β | Fold list into single value |
| CROSS | [α] × [β] → [(α,β)] | Cartesian product (for pairwise reasoning) |
| M | Σ* → Σ* | Neural oracle (base LLM) |

## Formal Guarantees

- **Theorem 1 (Termination):** λ-RLM is guaranteed to halt — every recursive call strictly reduces input size (rank)
- **Theorem 2 (Cost Bound):** Total cost T(n) is computable pre-execution
- **Theorem 4 (Optimal Partition):** k* = 2 minimizes cost under standard models
- **Corollary 5 (Scaling):** Direct LLM accuracy decays exponentially (Θ(ρ^{n/K})), λ-RLM decays only at power-law rate (Ω(n^{-c}))

## Empirical Results

### Accuracy (Macro-averaged %)
| Model Tier | Direct LLM | Normal RLM | λ-RLM | Gain vs RLM |
|-----------|:----------:|:----------:|:-----:|:----------:|
| Weak (8B/7B) | 6.1 | 13.8 | 35.7 | +21.9 |
| Medium (32B+) | 17.6 | 31.3 | 49.9 | +18.6 |
| Strong (235B+) | 25.0 | 50.1 | 58.9 | +8.8 |

### Latency (Seconds)
| Model Tier | Normal RLM | λ-RLM | Speedup |
|-----------|:---------:|:-----:|:-------:|
| Weak (8B/7B) | 248.8 | 61.2 | 4.1× |
| Medium (32B+) | 193.4 | 47.7 | 4.1× |
| Strong (235B+) | 128.8 | 38.8 | 3.3× |

### Key Findings
- **29/36 wins** across model-task comparisons (81% win rate)
- OOL-Pairs (O(n²) task): **+28.6 points**, **6.2× speedup** — quadratic cross-product handled symbolically
- Scale substitution: Qwen3-8B + λ-RLM matches Llama-70B + standard RLM; outperforms Llama-405B direct inference

## Models Tested
9 models across Qwen, Llama, Mistral families. Backends: NVIDIA NIM, Together AI.
