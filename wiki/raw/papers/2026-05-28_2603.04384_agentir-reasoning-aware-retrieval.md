# AgentIR: Reasoning-Aware Retrieval for Deep Research Agents

## Abstract

Deep Research agents are rapidly emerging as primary consumers of modern retrieval systems. Unlike human users who issue queries without documenting their intermediate thought processes, Deep Research agents generate explicit natural language reasoning before each search, revealing rich intent and context that existing retrievers entirely ignore. This paper introduces: (1) **Reasoning-Aware Retrieval**, a retrieval paradigm that jointly embeds the agent's reasoning trace alongside its query; and (2) **DR-Synth**, a data synthesis method that generates Deep Research retriever training data from standard QA datasets. The combination yields **AgentIR-4B**, a trained embedding model achieving **68% accuracy** on BrowseComp-Plus with Tongyi-DeepResearch, compared to 52% with conventional embedding models twice its size, and 37% with BM25.

---

## 1. Problem & Motivation

Deep Research agents (LLMs) perform multi-turn reasoning and searching (often 20+ turns). At each turn, the agent generates a **reasoning trace** (τ_t) before issuing a search query (q_t). This trace contains:

- **Task intent**: clarifies ambiguous queries (e.g., "backroom studio early 2010s euphoric" → looking for a Grammy-winning composer who made a progressive house track)
- **Reflection on prior results**: summarizes what's already known (e.g., "award X is Grammy")
- **Hypothetical search targets**: uses parametric knowledge conditioned on full history to narrow search space (e.g., country likely Sweden/Finland/Austria)

Existing retrievers throw this signal away — they embed only the final query q_t, treating agent queries like human queries. **AgentIR embeds R(τ_t, q_t)** instead of R(q_t).

---

## 2. Key Contributions

| Contribution | Description |
|---|---|
| **Reasoning-Aware Retrieval** | Jointly embed reasoning trace + query instead of query alone. Three signal types: task intent, prior-result reflection, hypothetical search targets. No extra LLM call — reasoning is "for free" from the agent loop. |
| **DR-Synth** | Synthesize training data for agent sub-queries from standard (Q, A, P) QA triples using agent rollouts + oracle reranking. |
| **AgentIR-4B** | Fine-tuned from Qwen3-Embedding-4B on 5,238 DR-Synth instances. +18% absolute accuracy over strong baselines, generalizes across agents without retraining. |

## 3. Methodology

### 3.1 Reasoning-Aware Retrieval

**Standard retrieval**: o_t ← R(q_t) — the retriever only sees the query string.

**Reasoning-aware**: o_t ← R(τ_t, q_t) — the retriever sees both the reasoning trace and the query.

**Embedding template** (Figure 5):
```
Instruction: Given a user's reasoning followed by a web search query, retrieve relevant
passages that answer the query while incorporating the user's reasoning.

Query: Reasoning: {reasoning} Query: {query}
```

### 3.2 DR-Synth: Training Data Construction

The core challenge: no training data exists for agent sub-queries. Existing QA datasets provide only global (Q, A, P) triples. DR-Synth converts these into sub-query-level instances.

**Pipeline** for a given (Q, A, P) tuple:

1. **Generate sub-queries**: Run Tongyi-DR agent on Q with a query-only retriever. Record (τ_t, q_t) for each search turn. Only keep trajectories that successfully answer Q (rejection sampling).
2. **Oracle reranking**: For each turn t:
   - Retrieve top-50 documents for q_t
   - Prepend positive documents P
   - Prompt LLM (GLM-4.6) to listwise rerank the pool using q_t, Q, and A
   - Label top-1 as positive d⁺_t, bottom-7 as hard negatives {d⁻_t}
3. **Train**: Contrastive loss (temperature 0.01) with in-batch negatives.
   - ℒ = −log[ exp(sim([τ_t, q_t], d⁺_t)/T) / (exp(sim([τ_t, q_t], d⁺_t)/T) + Σ exp(sim([τ_t, q_t], d⁻_t)/T)) ]

**Training data**: 5,238 instances from 250 WebShaper questions, ~1.15M document corpus.

### 3.3 AgentIR-4B

- **Backbone**: Qwen3-Embedding-4B, fine-tuned with LoRA, contrastive loss, Tevatron framework
- **Training data**: DR-Synth (5,238 instances from WebShaper)
- **Size**: 4B parameters — half the size of conventional 8B retrievers

## 4. Experiments

### 4.1 Setup

- **Benchmark**: BrowseComp-Plus (complex multi-hop, 20+ searches)
- **Agents tested**: Tongyi-DR, gpt-oss-120B, GLM-4.7 (zero-shot transfer)
- **Metrics**: End-to-end QA accuracy, recall of all retrieved documents, number of search calls
- **Retrieval**: Top-5 snippets truncated to 512 tokens

### 4.2 Main Results

| Agent | Retriever | Accuracy (%) | Recall (%) | Search Calls |
|-------|-----------|-------------|------------|--------------|
| Tongyi-DR | BM25 | 33.98 | 46.83 | 32.92 |
| Tongyi-DR | Qwen3-Embed-4B | 48.67 | 59.90 | 31.02 |
| Tongyi-DR | Qwen3-Embed-8B | 50.72 | 61.78 | 30.43 |
| Tongyi-DR | ReasonIR-8B | 51.03 | 63.62 | 31.15 |
| Tongyi-DR | Qwen3-Embed-4B + LLM Rerank | 55.66 | 68.35 | 28.85 |
| Tongyi-DR | **AgentIR-4B** | **66.27** | **78.86** | **25.91** |
| Tongyi-DR (visit) | **AgentIR-4B** | **68.07** | 76.58 | 24.49 + 3.41 Visit |
| oss-120b-high | **AgentIR-4B** | **66.99** | 78.13 | 24.08 |
| GLM-4.7 | **AgentIR-4B** | **61.18** | 73.10 | 23.85 |

### 4.3 Key Findings

1. **+18% absolute accuracy** over Qwen3-Embed-8B (50.72% → 66.27%) with half the parameters
2. **+10% over LLM reranking** (55.66% → 66.27%) — joint embedding beats 2-stage pipeline
3. **~20% fewer search calls** (32.92 → 25.91) — better retrieval reduces exploration overhead
4. **Cross-agent generalization**: Works with oss-120b and GLM-4.7 without retraining
5. **Reasoning trace matters**: Ablation shows reasoning trace signal is the primary driver of gains

### 4.4 Ablation: What Makes Reasoning Traces Valuable?

| Signal | Contribution |
|--------|-------------|
| Task Intent | Translates keyword queries into semantically precise retrieval targets |
| Reflection | Provides retrieval history context — avoids re-searching already-known answers |
| Hypotheses | Grounds HyDE-style speculation in actual agent context, more accurate than naive HyDE |

## 5. Significance

### 5.1 Relationship to Existing Work

- **Q2Q Reformulation** (Meng et al., 2026): Uses reasoning traces to translate keywords → natural language questions before retrieval. AgentIR goes further — jointly embedding reasoning + query trains the retriever to **internalize** the reformulation.
- **SID-1** (SID AI, 2025): RL-trained agentic retrieval model. AgentIR is complementary — it improves the retrieval backbone that even RL-trained agents depend on.
- **HyDE** (hypothetical document embeddings): AgentIR's reasoning traces ground speculation in the agent's actual search context, making it more accurate than standalone HyDE.

### 5.2 Implications

- **Free signal**: Reasoning traces cost nothing extra — the agent already generates them. AgentIR simply stops throwing them away.
- **Retriever co-design**: Retrieval models should be designed for agent consumers, not human consumers. This means training on agent-issued queries with their reasoning context.
- **Smaller models, better results**: AgentIR-4B (4B) outperforms 8B retrievers and LLM reranking pipelines. The architecture matters more than parameter count when the model is trained on the right signal.

### 5.3 Limitations

- DR-Synth requires running full agent rollouts for each training question — computationally expensive data synthesis
- Oracle reranking quality depends on the LLM judge (GLM-4.6 used)
- Only tested on BrowseComp-Plus; generalization to other benchmarks not yet demonstrated
- Single retriever type (bi-encoder); multi-vector (ColBERT) and learned sparse (SPLADE) variants not explored
