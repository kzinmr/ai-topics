# Superintelligent Retrieval Agent: The Next Frontier of Information Retrieval

**Authors:** Zeyu Yang, Qi Ma, Jason Chen, Anshumali Shrivastava
**Affiliations:** Meta Superintelligence Labs, Rice University (*work done at Meta)
**Date:** May 7, 2026
**arXiv:** [2605.06647](https://arxiv.org/abs/2605.06647)
**Code:** [facebookresearch/sira](https://github.com/facebookresearch/sira)

## Abstract

Current retrieval-augmented agents treat search as a multi-round black box: generate a query → inspect snippets → reformulate. This resembles a novice, not an expert. SIRA defines *superintelligence in retrieval* as the ability to compress multi-round exploration into a **single, corpus-discriminative retrieval action**.

- **Corpus-side:** an LLM enriches each document offline with missing search vocabulary.
- **Query-side:** the LLM predicts evidence vocabulary absent from the query, and document-frequency (DF) statistics filter proposed terms (absent, too common, or non-discriminative).
- **Final retrieval:** a single weighted BM25 call combining original query + validated expansion.

Across **10 BEIR benchmarks** and downstream QA tasks, SIRA **outperforms dense retrievers (E5), SPLADE, and state-of-the-art multi-round agentic baselines** — all while remaining **interpretable, training-free, and efficient**.

## Core Gap

An LLM may know what relevant evidence looks like, but it needs **index-visible statistics** and **explicit retrieval controls** to make that expectation discriminative against a whole corpus. SIRA bridges this gap by giving the LLM access to document frequency (DF) statistics from the BM25 inverted index.

## Pipeline

### Offline (once per corpus)
1. LLM reads each document, proposes missing search vocabulary (synonyms, abbreviations, alternate names)
2. DF filter prunes non-discriminative terms (too common: DF > τ·|C|)
3. Surviving terms decomposed into sliding-window n-grams and injected into BM25 index

### Online (per query)
1. LLM produces an **expected-response sketch** — concepts/entities/discriminative terms likely in answer but absent from query
2. DF filter prunes terms absent (DF=0) or too common
3. SIRA compiles a retrieval program conditioned on the validated sketch
4. Single weighted BM25 call: score(d) = BM25(q_orig, d) + w · BM25(q_exp, d)

### DF Filter
Shared by both enrichment stages:
- Query-side: Requires DF > 0 (term must exist in enriched index) and DF ≤ τ·|C| (not too common)
- Corpus-side: Only DF ≤ τ·|C| (terms are injected to exist)

## Experiments

- **10 BEIR benchmarks**: SIRA substantially outperforms dense retrievers (E5-unsupervised, E5-supervised, BGE), SPLADE, and multi-round agentic baselines (IRCoT, Search-R1)
- **Downstream QA**: With a pointwise LLM reranker, SIRA achieves SoTA on multiple QA tasks
- **Ablation**: Both corpus-side enrichment and query-side expansion contribute; DF filter is critical (without it, performance drops significantly)
- **Efficiency**: One-shot retrieval eliminates the multi-round overhead of agentic search

## Key Innovation

SIRA reframes retrieval as a **vocabulary alignment problem**: the gap between how users express queries and how documents describe answers. Rather than training embeddings or iterating through search results, it uses LLM knowledge to predict bridging vocabulary, validated by corpus statistics before retrieval executes. The result is a single, auditable BM25 query that outperforms substantially more expensive multi-round search.
