# Beyond Semantic Similarity: Rethinking Retrieval for Agentic Search via Direct Corpus Interaction

**arXiv**: [2605.05242](https://arxiv.org/abs/2605.05242) | **GitHub**: [DCI-Agent/DCI-Agent-Lite](https://github.com/DCI-Agent/DCI-Agent-Lite) (228 ⭐)
**Published**: May 3, 2026 | **HF Paper of the Day #2** (107 upvotes)

> **Tagline:** "The best retriever ... is no retriever" — replacing the entire agentic search pipeline (embedding model, vector index, top-k retrieval) with only `grep` and `bash`.

## Abstract

Modern retrieval systems expose a corpus through a fixed similarity interface that compresses access into a single top-k retrieval step before reasoning. This abstraction is efficient, but for agentic search, it becomes a bottleneck: exact lexical constraints, sparse clue conjunctions, local context checks, and multi-step hypothesis refinement are difficult to implement by calling a conventional retriever, and evidence filtered out early cannot be recovered by stronger downstream reasoning.

To tackle this limitation, the authors study **Direct Corpus Interaction (DCI)** — an agent searches the raw corpus directly with general-purpose terminal tools (grep, file reads, shell commands, lightweight scripts), without any embedding model, vector index, or retrieval API. No offline indexing required.

## Key Innovation

Instead of the conventional pipeline:

```
Query → Embedding Model → Vector Index → Top-k Retrieval → Agent Reasoning
```

DCI replaces it with:

```
Query → Agent → grep/find/bash/shell-pipelines → Raw Corpus → Agent Reasoning
```

The agent orchestrates its own search trajectory directly on raw text, using lightweight command-line tools — like a developer navigating a codebase.

## Method

### DCI-Agent-Lite (Minimal Scaffold)
A lightweight terminal coding agent adapted from Pi, restricted to raw terminal interaction. Accesses corpus through bash and file reads using grep, rg, and shell operations. Equipped with runtime context management for long-horizon exploration.

### DCI-Agent-ClaudeCode (CC)
Built on Claude Code (Sonnet 4.6), with web-search, web-fetch, and subagents disabled. Provides a mature coding agent harness for DCI.

### Runtime Context Management
Five levels of context management to support long-horizon search:
| Level | Truncation | Compaction | Summarization |
|-------|-----------|------------|---------------|
| L0 | ✗ | ✗ | ✗ |
| L1 | ✓ (max 50K chars) | ✗ | ✗ |
| L2 | ✓ (max 20K chars) | ✗ | ✗ |
| L3 | ✓ (max 20K chars) | ✓ | ✗ |
| L4 | ✓ (max 20K chars) | ✓ | ✓ |

L3 achieves the best accuracy despite retaining less gold evidence than L1 — indicating that selective discarding of old observations is beneficial.

## Key Results

### Agentic Search (BrowseComp-Plus)
With the same Sonnet 4.6 backbone, replacing Qwen3-Embedding-8B with DCI:
- **Accuracy**: 69.0% → 80.0% (**+11.0 points**)
- **Cost**: $1,440 → $1,016 (**−29.4%**)

### Multi-Hop QA
DCI-Agent-CC achieves **83.0 average accuracy**, surpassing the strongest retrieval-agent baseline (ASearcher-Local-14B) by **+30.7 points**.

### IR Ranking (BRIGHT, BEIR)
DCI-Agent-CC reaches **68.5 average NDCG@10**, outperforming the best retrieval baseline (ReasonRank) by **+21.5 points**.

### Generality
DCI-Agent-Lite (minimal Pi-based harness, GPT-5.4 nano) also consistently outperforms conventional retrievers, demonstrating that the DCI interface itself — not the specific harness — drives the gains.

## Ablation Studies (6 RQs)

### RQ2: Why does DCI help?
DCI's advantage comes less from higher gold-document recall and more from **fine-grained discovery, composition, and use of evidence** through flexible, compositional bash commands. Among 176 BrowseComp-Plus questions that DCI-Agent-CC answers correctly but the retrieval agent misses, only 34 contain no gold documents retrieved by the conventional retriever — DCI enables better evidence *utilization*, not just better recall.

### RQ3: Evidence utilization patterns
DCI agents exhibit qualitatively different search patterns: they issue more narrowly-targeted follow-up queries grounded in localized evidence extracted from prior reads, rather than broad similarity-based retrieval.

### RQ4: Corpus scale robustness
DCI maintains performance across varying corpus sizes without requiring index rebuilds.

### RQ5: Context management impact
Aggressive context management is non-monotonic — L3 (truncation + compaction) achieves best accuracy, while more aggressive L4 (adding summarization) degrades performance.

### RQ6: Tool usage patterns
The dominant bash tool decomposes into ~10 concrete command intents (grep patterns, find, file reads, pipelines), showing that DCI agents develop compositional search strategies rather than using tools naively.

## Broader Implications

The paper argues that as language agents become stronger, retrieval quality depends not only on reasoning ability but also on the **resolution of the interface** through which the model interacts with the corpus. DCI opens a broader interface-design space for agentic search — the "medium" (McLuhan) of retrieval shapes what agents can discover.

The approach requires **zero preprocessing**, adapts immediately to changing local corpora, and works with any terminal-equipped agent harness.

## Connections to Prior Work

- **Coding Agents as Retrieval/Processing** (Cao et al., 2026): Precedent for using coding agents on file-system-organized text for BrowseComp-Plus. DCI generalizes this to arbitrary corpora without file-system pre-organization.
- **Agentic Search** (Meng et al., 2026): Identified query mismatch between agent-issued keywords and dense retrievers. DCI sidesteps this entirely by removing the retriever.
- **SID-1** (SID AI, 2025): RL-trained agentic retriever that still uses a retrieval backend. DCI eliminates the backend entirely.
- **Q2Q Reformulation** and **AgentIR**: Techniques to bridge agent queries to dense retrievers. DCI obviates the need for such bridging.
