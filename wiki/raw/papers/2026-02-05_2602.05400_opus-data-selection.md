# OPUS: Towards Efficient and Principled Data Selection in Large Language Model Pre-training in Every Iteration

**arXiv:** [2602.05400](https://arxiv.org/abs/2602.05400) | **Date:** 2026-02-05

## Abstract

As high-quality public text approaches exhaustion, a phenomenon known as the **Data Wall** (Villalobos et al., 2022), pre-training is shifting from "more tokens" to "better tokens." However, existing methods either rely on heuristic static filters that ignore training dynamics, or use dynamic yet optimizer-agnostic criteria based on raw gradients.

OPUS (Optimizer-induced Projected Utility Selection) is a dynamic data selection framework that defines utility in the **optimizer-induced update space**. OPUS scores candidates by projecting their effective updates, shaped by modern optimizers, onto a target direction derived from a stable, in-distribution proxy. To ensure scalability, OPUS employs **Ghost technique** with **CountSketch** for computational efficiency, and **Boltzmann sampling** for data diversity, incurring only **4.7% additional compute overhead**.

## Key Results

- **GPT-2 Large/XL on FineWeb/FineWeb-Edu (30B tokens):** OPUS outperforms industrial-level baselines and even **full 200B-token training**.
- **Continued pre-training of Qwen3-8B-Base on SciencePedia:** Achieves superior performance using only **0.5B tokens** compared to full training with 3B tokens (6x data efficiency).
- Works across diverse corpora, quality tiers, optimizers (AdamW, Muon), and model scales.
- Combined with industrial static filters, OPUS further improves pre-training efficiency even with lower-quality data.

## Method Overview

### Core Insight
A batch is valuable only insofar as it moves parameters in a direction that improves the model's performance on a high-quality target distribution under the **optimizer's specific geometry**. OPUS eliminates the discrepancy between scoring and training that arises when AdamW/Muon training is treated as if it were SGD.

### Key Components

1. **Optimizer-Aware Utility:** Closed-form approximations for effective update directions of AdamW and Muon. Scores candidates in actual optimizer-induced geometry, not raw gradients.

2. **Bench-Proxy:** Builds a proxy pool by retrieving benchmark-aligned documents from the pre-training corpus using a frozen sentence encoder (Arctic-Embed-L v2). Provides a reliable, in-distribution target direction (~30M token budget).

3. **Scalable Estimation (Ghost + CountSketch):**
   - Ghost technique: rank-1 outer-product gradients (avoids materializing full per-sample gradients)
   - CountSketch: projects into low-dimensional space (ℝ^m) for efficient inner products
   - Combined overhead: ~4.7%

4. **Boltzmann Sampling with Redundancy Penalty:**
   - Soft sampling prevents diversity collapse
   - In-step penalty discourages selecting redundant samples
   - Robust to noisy proxy estimates

### Utility Definition
$$U^{(t)}(S) := \mathcal{L}(D_{val}; \theta_t) - \mathcal{L}(D_{val}; \theta_{t+1}(S))$$

The marginal gain of adding sample $z$ is approximated as:
$$U_z^{(t)} \approx \eta_t \langle \mathbf{u}_z^{(t)}, \nabla_\theta \mathcal{L}(D_{val}; \tilde{\theta}_t(\hat{B}_t)) \rangle$$

where $\mathbf{u}_z^{(t)} = P_t \nabla_\theta \mathcal{L}(z; \theta_t)$ is the optimizer-preconditioned update direction, $P_t$ being the preconditioner from AdamW/Muon.

### Bench-Proxy Construction
1. Embed target benchmark validation set and pre-training documents using Arctic-Embed-L v2 (frozen)
2. Compute cosine similarity; max-similarity aggregation per document
3. Accumulate highest-scoring documents up to token budget (e.g., 30M tokens)
4. During training, randomly sample proxy batch from this pool

## Significance

OPUS represents a shift from **static, heuristic data filtering** (FineWeb-Edu, DCLM) to **dynamic, optimizer-aware data selection** that adapts to the model's evolving needs at each training step. The framework bridges the gap between data selection theory and modern optimizer practice (AdamW, Muon), and demonstrates that principled selection can achieve massive data efficiency gains (6x fewer tokens in continued pre-training).

The 4.7% overhead makes it practical for large-scale pre-training, and the approach generalizes across model scales, data quality tiers, and optimizer types.
