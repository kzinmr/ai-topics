# Coding Agents are Effective Long-Context Processors

**Authors:** Weili Cao, Xunjian Yin, Bhuwan Dhingra, Shuyan Zhou
**Date:** March 20, 2026
**Venue:** arXiv.org — preprint, no peer-reviewed venue

> **Blocking Note:** arXiv-only, no peer-reviewed venue. Explicitly requested by user for wiki ingestion.

## Abstract

LLMs struggle to process long contexts even when they can access them, suffering from "context rot." This research demonstrates that coding agents (OpenAI Codex, Claude Code) are superior long-context processors because they externalize reasoning into explicit, executable interactions. By organizing text into directory structures, agents use terminal commands (`grep`, `sed`, `ripgrep`) and Python scripts to navigate and manipulate data, scaling to 3 trillion tokens.

## Key Findings

### Performance vs SOTA
Coding agents outperformed existing SOTA by **17.3%** on average across 4 of 5 major benchmarks:

| Benchmark | Context | Agent Score | Prev SOTA | Δ |
|-----------|---------|------------|-----------|--|
| **BrowseComp-Plus** | 750M tokens | **88.50** | 80.00 | +10.6% |
| **Oolong-Syn** | 536K tokens | **71.75** | 64.38 | +11.4% |
| **Oolong-Real** | 385K tokens | **37.46** | 24.09 | +55.5% |
| **NQ (Natural Qs)** | 3T tokens | **56.00** | 50.90 | +10.0% |
| **LongBench** | 188K tokens | 62.50 | 63.30 | Competitive |

### Core Mechanisms
1. **Native Tool Proficiency** — Index/slice with `nl`+`sed`, programmatic search with Python+regex
2. **File System Familiarity** — Folder structure (89.0%) outperforms single file (83.0%) due to coding agents' inductive priors from code repository training

### Negative Result on Retrieval Tools
> "Equipping coding agents with retrieval tools does not consistently improve performance and can even degrade it — standard retrievers become the default discovery mechanism and displace broader file-system exploration strategies."

### Emergent Strategies
- **Multi-hop QA**: Iterative "search-and-refine" loop across 6-hop chains
- **Analytical Tasks**: Abandons search for Python scripts to parse/count/sort
- **Reading Comprehension**: Balanced tool usage with LLM reasoning

### Cost
- BrowseComp-Plus: ~$0.70/query (coding agent) vs ~$0.27 (GPT-5 full context, which couldn't see the full corpus)
- Oolong-Syn: ~$0.19 (coding agent) vs ~$1.42 (GPT-5 full context)
