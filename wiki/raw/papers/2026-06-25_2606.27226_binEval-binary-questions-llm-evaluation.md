## Abstract

Evaluating LLM outputs remains a major bottleneck in NLP: human evaluation is expensive and slow, lexical metrics correlate poorly with human judgments on open-ended generation, and holistic LLM judges often produce opaque scores that are hard to debug. We propose **BINEVAL**, a framework that decomposes evaluation criteria into atomic binary questions and aggregates the resulting verdicts into interpretable, multi-dimensional scores. Given a task prompt, a meta-prompt generates fine-grained evaluation questions, and an LLM answers them independently for each output, yielding transparent question-level feedback together with calibrated overall scores. This decomposition makes evaluation easier to inspect, easier to diagnose, and directly usable for prompt improvement. Across SummEval, Topical-Chat, and QAGS, BINEVAL matches or outperforms strong baselines including UniEval and G-Eval, with especially strong results on factual consistency benchmarks such as QAGS. Beyond competitive correlation with human judgments, BINEVAL better matches human score distributions and avoids the ceiling effects common in prior LLM judges, leading to better discrimination between borderline and clearly flawed outputs. We further show that the same question-level feedback supports iterative prompt optimization, improving evaluator prompts on summarization and generation prompts on IFBench under both self-update and cross-model update settings. Overall, BINEVAL provides a task-agnostic, training-free, and interpretable evaluation framework that combines strong empirical performance with practical diagnostic and optimization value.

## Key Contributions

1. **General framework for interpretable evaluation** — Decomposes evaluation criteria into atomic yes/no questions, yielding a task-agnostic and modular method.
2. **Strong performance without task-specific training** — Matches or exceeds trained evaluators (UniEval) and holistic LLM judges (G-Eval) on SummEval, Topical-Chat, and QAGS.
3. **Iterative prompt improvement** — Two-phase optimization loop that improves prompts for both summarization and IFBench.
4. **Debuggable scores** — Each score is grounded in individual verdicts with explanations.

## Method

### BINEVAL Framework (3 Steps)

1. **Summarize** — Task prompt T is summarized into explicit requirements R = {r1, r2, ..., rK}.
2. **Decompose** — Each requirement rk is decomposed into binary questions (yes/no). Questions are organized into evaluation dimensions D (coherence, consistency, fluency, relevance). Questions partition as Q = ∪_{d∈D} Qd.
3. **Evaluate** — An evaluator LLM E answers each question independently, producing binary verdicts f_E(x, y, qi) ∈ {0, 1} with natural-language explanations.

### Scoring

- Per-dimension: S_d(x, y) = (1/|Q_d|) Σ f_E(x, y, qi)
- Overall: S(x, y) = (1/N) Σ f_E(x, y, qi)
- Scores in [0, 1], affinely scalable to any target interval.

### Cross-Model Prompt Update

Disagreements between source and target evaluators on specific binary questions provide fine-grained signal for improvement. Loop until scores align within tolerance ε.

### Self Prompt Update

Iteratively improves a generator by using evaluator-identified failures as feedback. Extracts lessons from failing questions, deduplicates semantically, and rewrites the prompt.

## Experimental Results

### Part I: Evaluation Quality (SummEval)

| Method | Coherence | Consistency | Fluency | Relevance | Average |
|--------|-----------|-------------|---------|-----------|---------|
| ROUGE-1 | .167/.126 | .160/.130 | .115/.094 | .326/.252 | .192/.150 |
| BERTScore | .284/.211 | .110/.090 | .193/.158 | .312/.243 | .225/.175 |
| BARTScore | .448/.342 | .382/.315 | .356/.292 | .356/.273 | .385/.305 |
| UniEval (T5) | .575/.442 | .446/.371 | .449/.371 | .426/.325 | .474/.377 |
| G-Eval (GPT-4) | .582/.457 | .507/.425 | .506/.455 | .547/.433 | .514/.418 |
| **BINEVAL (Claude)** | **.652/.541** | **.655/.615** | .540/.470 | .404/.339 | **.563/.491** |

Spearman ρ / Kendall τ. BINEVAL (Claude) achieves best average correlation, especially strong on consistency (factual quality).

### Part II: QAGS (Factual Consistency)

BINEVAL achieves Spearman correlation 0.620 on QAGS, substantially outperforming G-Eval and UniEval. Decomposing factual consistency into targeted questions is especially effective for hallucination-prone data (XSum).

### Iterative Prompt Update (SummEval)

| Dimension | Self-Update Δ | Cross-Model Δ |
|-----------|--------------|---------------|
| Coherence | +.089 | +.070 |
| Consistency | +.091 | +.136 |
| Fluency | +.119 | +.072 |
| Relevance | .000 | .000 |
| Average | +.075 | +.070 |

Self-update helps most on coherence/fluency; cross-model update helps most on consistency. Most gains appear within first 1–2 iterations.

### IFBench (Generation Prompt Update)

Self-update achieves +3.4pp on strict accuracy (38.0% at iteration 3). Format and sentence constraints improve by 17pp each, but count/ratio/words/repeat constraints show little improvement (computational constraints not solvable by prompt refinement alone).

## Key Insights

- **Multiple binary questions > single holistic score**: Decomposing into targeted questions provides more robust and transferable evaluation signal.
- **Variance reduction via aggregation**: Aggregating N weakly correlated binary classifiers reduces variance proportionally to 1/N.
- **Coverage of failure modes**: Decomposition forces explicit enumeration of criteria, improving recall over holistic judgments.
- **Ceiling effect avoidance**: BINEVAL better matches human score distributions and discriminates between borderline and clearly flawed outputs.
- **Relevance is harder to decompose**: G-Eval (GPT-4) remains best on relevance, indicating some broader semantic judgments are harder to capture with binary decomposition.
- **Computational constraints resist prompt optimization**: Count, ratio, words, repeat constraints in IFBench show little improvement — these require precise computation during generation, not clearer instructions.

## Baselines Compared

- **UniEval** (Zhong et al., 2022) — T5-based, reformulates evaluation as Boolean QA, fine-tuned per dimension.
- **G-Eval** (Liu et al., 2023) — Chain-of-thought + Likert-scale rating with GPT-4.
- **Prometheus 2** (Kim et al., 2024) — Open-source evaluator approximating human/proprietary judgments.
- **AlpacaEval** (Li et al., 2023) — Pairwise preference judgments.
- **MT-Bench / Chatbot Arena** (Zheng et al., 2023) — Preference-based judgments.
- Lexical metrics: ROUGE, BLEU, BERTScore, MoverScore, BARTScore.
