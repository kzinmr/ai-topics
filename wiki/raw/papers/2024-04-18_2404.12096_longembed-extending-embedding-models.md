# LongEmbed: Extending Embedding Models for Long Context Retrieval

**arXiv: 2404.12096v3 (Nov 2024)**
**Code & Models:** https://github.com/dwzhu-pku/LongEmbed

## Abstract

Embedding models play a pivotal role in modern NLP applications such as IR and RAG. While the context limit of LLMs has been pushed beyond 1 million tokens, embedding models are still confined to a narrow context window not exceeding 8K tokens, refrained from application scenarios requiring long inputs such as legal contracts. This paper explores context window extension of existing embedding models, pushing the limit to 32K without requiring additional training. First, we examine the performance of current embedding models for long context retrieval on our newly constructed LongEmbed benchmark. LongEmbed comprises two synthetic tasks and four carefully chosen real-world tasks, featuring documents of varying length and dispersed target information. Benchmarking results underscore huge room for improvement in these models.

## Key Contributions

1. **LongEmbed benchmark** — 2 synthetic + 4 real-world tasks with documents up to 32K tokens and uniformly dispersed target information
2. **Training-free extension** — position interpolation, NTK, SelfExtend can extend context by several folds without retraining
3. **RoPE superiority** — RoPE-based embedding models significantly outperform APE models for context window extension
4. **Released models**: E5-Base-4k and E5-RoPE-Base

## Motivation

- Most embedding models (E5, BGE, GTE) support only **512 tokens**
- Training from scratch is expensive (BGE-M3 used 96 A100 GPUs)
- Goal: extend existing models without full retraining, inspired by LLM context extension techniques

## LongEmbed Benchmark

### Design Principles
- Long documents (up to 32K tokens)
- Uniformly dispersed target information (prevents header/locality shortcuts)

### Dataset Overview

| **Synthetic** | | | | | |
| Passkey | — | 400 | 800 | 11 | varying (0.25K–32K ×0.75K) |

- **Passkey**: personalized passkeys in garbage sentences; test document-level retrieval
- **Needle**: facts embedded in long essays (GPT-4 generated facts inserted into PaulGrahamEssay)
- **Metrics**: nDCG@10 for real tasks, Acc@1 for synthetic tasks

### Limitations of Prior Benchmarks
- **BEIR**: documents average <300 words → too short
- **LoCo**: biased information distribution; even E5-Base (512 ctx) scores >85% nDCG@10 on 3/8 tasks

## Context Window Extension Methods

### For APE Models (e.g., E5, GTE)

1. **Parallel Context Windows (PCW)**: chunk, encode separately, average embeddings
2. **Grouped Positions (GP)**: map `pos_id → ⌊pos_id/s⌋`
3. **Recurrent Positions (RP)**: map `pos_id → pos_id mod L₀`
4. **Linear Position Interpolation (PI)**: map `pos_id → pos_id/s`, linearly interpolate new position embeddings

### For RoPE Models (e.g., E5-RoPE, E5-Mistral)

5. **SelfExtend (SE)**: group larger distances as `⌊(m-n)/w⌋`, but keep normal relative positions within neighbour window `w`
6. **NTK-Aware Interpolation (NTK)**: scale RoPE frequencies non-uniformly: `θ' = (10000λ)^{-2j/d}`, with `λ` slightly > scaling factor `s`

### Further Tuning (APE models only)
- Freeze original position embeddings; tune only newly added position vectors
- Simulate long sequences via **skipping bias** `u` (sampled uniformly from `[0, L_train - L_original]`)

## Key Findings

### Benchmarking Results
- **Baseline models degrade severely** beyond their training context length
- E5-Base (512 ctx): strong at 512 tokens, drops sharply afterward
- BGE-en-icl (8K ctx): holds well to ~8K, then degrades
- No model maintains high performance at 32K without extension

### Extension Results
| Method | E5-Base (APE, 512 ctx) | E5-RoPE-Base (RoPE, 512 ctx) |
|--------|------------------------|------------------------------|
| Baseline | Degrades >512 | Degrades >512 |
| Position Interpolation | Extends to ~2K | — |
| NTK | — | Extends to 32K+ |
| SelfExtend | — | Strong to 32K |
| PCW + avg pooling | Moderate gain | Moderate gain |

### RoPE vs APE
- **RoPE fundamentally superior** for context extension — relative position encoding via `(m-n)θ` generalizes naturally
- APE models hit a ceiling; further tuning helps but gap remains
- SelfExtend with neighborhood window `w` gives best RoPE results

## Relationship to Embedding Long-Context Degradation

This paper (Apr 2024) presents **solutions** to the problem later quantified by [[concepts/embedding-long-context-degradation|Jina AI's blindness study]] (Mar 2025):

| Aspect | LongEmbed (2024) | Jina AI Blindness (2025) |
|--------|-----------------|-------------------------|
| **Focus** | How to extend context | How embeddings fail at scale |
| **Context range** | 512 → 32K | 128 → 8K |
| **Key finding** | PI/NTK/SelfExtend works | Even 8K-capable models degrade at ~4K |
| **Gap** | Passkey/NIAH tasks | Semantic one-hop NIAH (harder) |

**Synthesis**: LongEmbed shows that RoPE-based models can be extended to 32K. But the Jina study reveals that **even after extension**, semantic retrieval (not just lexical matching) collapses at ~4K tokens. The two papers together define the current frontier: **position encoding is solvable, semantic focus is not**.
