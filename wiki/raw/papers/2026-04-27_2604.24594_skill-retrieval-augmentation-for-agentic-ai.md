# Skill Retrieval Augmentation for Agentic AI

> **INGEST NOTE**: This paper is arXiv-only with no peer-reviewed venue found. It was ingested per user override (kzinmr, 2026-04-30). User comment: "Progressive disclosure and skill is a big part of agent harness engineering. Effective retrieval over skills is going to be big, might even become 'web scale'." — jobergum

## Abstract

Traditional AI agents rely on in-context skill injection, listing all available skills in the prompt. This fails as skill libraries grow — context budgets are consumed and model accuracy degrades. The authors propose **Skill Retrieval Augmentation (SRA)**, where agents dynamically retrieve, incorporate, and apply skills from a large external corpus on demand. They introduce **SRA-Bench** (5,400 tasks, 26,262 skills) to evaluate the pipeline and find that while retrieval improves performance, a critical "incorporation bottleneck" remains: current LLMs struggle to distinguish between gold skills and distractors once retrieved.

## Key Findings

- **SRA improves performance** over LLM Direct (parametric only), but remains below Oracle (perfect skill) upper bound
- **Qwen3-235B**: 53.3% (Direct) → 62.8% (SRA) vs 67.5% (Oracle)
- **Llama-3.3-70B**: 47.8% (Direct) → 59.2% (SRA) vs 64.4% (Oracle)
- **Incorporation bottleneck**: Agents load skills at similar rates regardless of whether a gold skill was retrieved
- **Progressive Disclosure** (load after decision) outperforms Full-Skill Injection (load everything upfront)
- **BM25 + LLM reranking** provides the most consistent end-to-end gains

## Architecture: Three-Stage SRA Pipeline

1. **Skill Retrieval** — Identify relevant skills from a massive corpus (26K+ skills)
2. **Skill Incorporation** — Decide which retrieved skills to "load" into active context
3. **Skill Application** — Execute the skill to solve the task

## Research Agenda

1. Structured skill libraries (graphs/hierarchies not flat lists)
2. Utility-aware retrieval (optimize for helpfulness, not semantics)
3. Skill refinement (offline debugging loops)
4. Parametric skill augmentation (convert frequent skills to LoRAs/adapters)
