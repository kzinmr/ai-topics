# Self-Distilled Agentic Reinforcement Learning (SDAR)

## Abstract

RL is central for post-training LLM agents, but trajectory-level reward provides only coarse supervision for long-horizon interaction. On-Policy Self-Distillation (OPSD) complements RL with dense token-level guidance from a teacher branch augmented with privileged context. However, transferring OPSD to multi-turn agents is problematic due to compounding multi-turn instability and the need for asymmetric treatment of skill-conditioned privileged guidance.

SDAR treats OPSD as a **gated auxiliary objective** while keeping RL as the primary optimization backbone. It maps detached token-level signals into a sigmoid gate that strengthens distillation on teacher-endorsed positive-gap tokens and softly attenuates negative teacher rejections.

Key results (vs GRPO baseline, Qwen2.5/Qwen3 families):
- **ALFWorld**: +9.4% (3B), +4.7% (7B)
- **Search-QA**: +7.0% (both 3B and 7B)
- **WebShop Acc**: +10.2% (7B)
- Avoids catastrophic instability of naïve GRPO+OPSD
- Consistently outperforms hybrid RL–OPSD methods (Skill-SD, RLSD)

## 1. Problem & Observations

### Two Paradigms for Agent Training
- **RL**: Task-level optimization via environment/verifier feedback (GRPO).
- **On-Policy Self-Distillation (OPSD)**: Dense token-level guidance from a teacher branch with privileged context (e.g., retrieved skills).

### Observation 1: Multi-turn OPSD Instability
Once the student agent drifts from the teacher-supported trajectory, token-level supervision becomes unreliable. Leads to surging per-turn KL divergence and catastrophic degradation. Naïve GRPO+OPSD fails; previous attempts (TCOD) use rigid schedules.

### Observation 2: Asymmetric Trust in Privileged Guidance
- Teacher branch = same policy + privileged context (skills) → not independently stronger
- **Positive gap** (teacher prob > student prob): endorsement signal → safe for strong distillation
- **Negative gap** (teacher prob < student prob): possibly from poor skill quality, failure to utilize skills, or multi-turn drift
- Over 50% of tokens have negative gaps → pervasive issue
- Design philosophy: trust positive endorsements, treat negative rejections conservatively

## 2. Method: SDAR

$$\\mathcal{L}_{\\text{SDAR}} = \\mathcal{L}_{\\text{RL}} + \\lambda_{\\text{SDAR}} \\cdot \\mathcal{L}_{\\text{SDAR}}^{\\text{aux}}$$

where $\\mathcal{L}_{\\text{RL}}$ is the standard GRPO loss (untouched).

### Token-Level Gap

$$\\Delta_t = \\log \\pi_T(y_t \\mid s_t^+) - \\log \\pi_\\theta(y_t \\mid s_t)$$

### Sigmoid Gate

$$g_t = \\sigma(\\beta \\cdot \\text{signal}_t) \\in (0,1)$$

Three gating strategies:
1. **Entropy gating**: $g_t = \\sigma(\\beta \\cdot h_t)$ — targets high-uncertainty tokens
2. **Gap gating** (default): $g_t = \\sigma(\\beta \\cdot \\Delta_t)$ — amplifies positive-gap, attenuates negative-gap tokens
3. **Soft-OR gating**: $g_t = \\sigma(\\beta[1 - (1-h_t)(1-\\Delta_t)])$ — combines both signals

Gate is detached (sg), gradients flow only through student log-prob:

$$\\ell_t^{\\text{SDAR}} = g_t \\cdot (\\log \\pi_T(y_t|s_t^+) - \\log \\pi_\\theta(y_t|s_t))$$

### Theoretical Properties
- Equivalent to weighted log-likelihood (maximizing $\\operatorname{Agg}(g_t \\log \\pi_\\theta(y_t|s_t))$)
- Gradient bounded by $g_t \\in (0,1)$ → no amplification of unreliable signals
- Gate is strictly increasing in $\\Delta_t$ → provides smooth, self-paced curriculum
- RL advantage gradient remains unbiased (L_RL untouched)

## 3. Experiments

### Setup
- **Benchmarks**: ALFWorld (6 household tasks, 3,827 tasks), Search-QA (7 datasets: NQ, HotpotQA, etc.), WebShop (128 fixed validation tasks)
- **Base models**: Qwen2.5-3B/7B-Instruct, Qwen3-1.7B-Instruct
- **Training**: 150 steps, 8×H800 GPUs, λ_SDAR=0.01, β=5.0, Skills from SkillBank (Qwen2.5-3B-Instruct)
- **Baselines**: Vanilla, Skill-Prompt, OPSD-only, GRPO, Skill-GRPO (±test skills), GRPO+OPSD (naïve), Skill-SD, RLSD

### Main Results

| Model | ALFWorld (%) | Search-QA (avg %) | WebShop Score/Acc |
|-------|-------------|-------------------|-------------------|
| Qwen2.5-3B + SDAR | **84.4** (+9.4 vs GRPO 75.0) | **43.4** (+7.0 vs 36.4) | 85.0/68.0 (+5.3/+4.7) |
| Qwen2.5-7B + SDAR | **85.9** (+4.7 vs 81.2) | **49.0** (+7.0 vs 42.0) | **89.4/82.8** (+8.5/+10.2) |
| Qwen3-1.7B + SDAR | **77.7** vs GRPO+OPSD 37.9 (collapse) | **41.2** vs GRPO+OPSD 9.5 (collapse) | **75.4/61.7** |

Key findings:
- GRPO+OPSD **collapses** on smallest model (1.7B) — SDAR avoids this
- Gains hold across model scales (1.7B → 7B)
- SDAR > Skill-SD and RLSD consistently

### Ablations
- Entropy gating alone fails to discriminate quality of retrieved skill knowledge → gap gating is critical
- β sensitivity: performance stable over β ∈ [2,10]
- λ sensitivity: robust across λ ∈ [0.001, 0.1]
- Without gate (GRPO+OPSD): high KL divergence surge, token-level signal becomes counterproductive
- Gate analysis: positive-gap tokens get weight near 1.0; negative-gap tokens near 0.0 → gating is asymmetric as designed

### Per-Turn Analysis
Traditional OPSD/GRPO+OPSD: KL divergence surges with turn count (T=4: ~10× T=1 level)
SDAR: KL divergence stays flat across all turn counts — **critical for long-horizon agents**

## 5. Significance

SDAR demonstrates that the key to stable multi-turn agent RL training is not choosing between RL and distillation, but giving each token the autonomy to decide how much distillation it needs. The sigmoid gate provides a self-paced curriculum — no hyperparameter schedule, no human-designed heuristics — just the model's own confidence signals driving the learning process.
