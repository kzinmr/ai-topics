# Vibe coding and agentic engineering are getting closer than I'd like

Simon Willison reflects on Heavybit's High Leverage podcast (Ep.9 with Joseph Ruscio), sharing his realization that vibe coding and agentic engineering have started to converge in his own work.

## Key Insight: The Blurring Line

Simon's original definition:
- Vibe coding = not looking at code at all, not caring about code quality, fine for personal tools
- Agentic engineering = professional engineers using agents to amplify expertise, building higher quality

But now: "as the coding agents get more reliable, I'm not reviewing every line of code that they write anymore, even for my production level stuff."

## The Guilt

"I know full well that if you ask Claude Code to build a JSON API endpoint... it's just going to do it right... But I'm not reviewing that code. And now I've got that feeling of guilt: if I haven't reviewed the code, is it really responsible for me to use this in production?"

## Resolution: Trust as a Team Analogy

Compares to working at larger organizations where other teams build services his team depends on. He doesn't read every line of their code — he trusts them based on reputation, and only digs in when problems arise. Starting to treat agents the same way, "but it still feels uncomfortable, because human beings are accountable for what they do."

"Claude Code does not have a professional reputation! It can't take accountability for what it's done. But it's been proving itself anyway."
