# Codex Mobile Power User Guide

**Author**: @dimillian
**Date**: June 14, 2026
**Source**: X/Twitter (quoted tweet)

## Core Thesis

Your phone is a **control plane** for work running on your Mac / Windows machine / devbox.

## Best Workflows

1. **Start threads on the right host + workspace** — use mobile to kick off work on the correct machine
2. **Pick branch + worktree from mobile** — select your working context before the task starts
3. **Run local environment setup before the task starts** — prepare the environment so Codex can work immediately
4. **Use mobile as a launchpad** — the actual heavy lifting happens on the desktop/devbox, not the phone

## Key Insight

The pattern treats mobile as a **remote control** rather than a development environment. This inverts the typical assumption that coding agents need to run locally on the device you're holding.

> "Your phone is a control plane for work running on your Mac / Windows machine / devbox"

This is relevant to [[concepts/codex]] workflows and the broader pattern of **distributed agent orchestration** where different devices play different roles in the same workflow.
