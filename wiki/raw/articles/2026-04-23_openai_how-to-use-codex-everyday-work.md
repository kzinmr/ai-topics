# How to Use Codex for Everyday Work

**Source:** OpenAI Academy
**URL:** https://openai.com/academy/how-to-use-codex-for-everyday-work
**Published:** April 23, 2026

---

ChatGPT Codex transforms scattered work materials—calendars, messages, emails, docs, dashboards, spreadsheets, trackers, decks—into review-ready artifacts. Instead of prompting from scratch, feed it your team's existing context and ask for the first usable version.

---

## 1. Create a Daily Work Brief

**Use this when:** You need a single clear daily brief turning calendar, message, email, and follow-up context into priorities and action items.

**What you bring:** Calendar, unread messages, unread email, open follow-ups, notes, priority context.
**What Codex returns:** Daily work brief with priorities, meeting prep, reply needs, decisions owed, FYIs, action flags.
**Suggested plugins:** Google Calendar, Gmail, Slack, Google Drive, Documents.

### Starter prompt
> Set up a weekday work brief that starts in the morning and keeps checking throughout the day. At the start of the day, review today's calendar, unread direct messages and mentions from the last 24 hours, unread email from the last 24 hours, my running list of open follow-ups, and any recent context that affects today's priorities. Create a short brief with priorities, meeting prep, messages that need replies, decisions I owe, and useful FYIs. Then check back every hour until the end of the workday for new replies, meeting changes, or follow-ups that need attention. Only update me when something changes or needs action.

## 2. Weekly Summary

**Use this when:** You need a manager-ready update from a week of work without reconstructing everything from memory.

**What you bring:** Calendar, edited docs, sent messages, planning tracker, project notes, relevant weekly context.
**What Codex returns:** Weekly update with completed work, decisions, changes, blockers, follow-ups, next priorities, source links.

### Starter prompt
> I'm writing my weekly update. Use my calendar, documents I edited, messages I sent in work channels, my main tracker or planning doc, and anything else that looks relevant to this week. Write a manager-ready summary that covers work completed, decisions made, important changes, blockers, follow-ups, and next week's priorities. Include source links where possible, and separate confirmed facts from anything that is an inference.

---

## 3. Draft Slide Decks

**Use this when:** You have source material for a presentation but need an editable deck with structure, speaker notes, and layout fixes.

**What you bring:** Project brief, source docs, metrics, customer/audience context, slide template, review expectations.
**What Codex returns:** Editable draft deck with slide structure, speaker notes, charts/visuals, layout fixes, missing-data flags.
**Suggested plugins:** Google Drive, Presentations, Documents, Figma, Canva.

---

## 4. Decision Memos

**Use this when:** You need a structured decision memo from scattered background material.

**What you bring:** Problem context, input from stakeholders, data/analysis, constraints, timeline.
**What Codex returns:** Decision memo with problem statement, options analysis, recommendation, risks, implementation plan.

---

## 5. Launch Kits

**Use this when:** You need a coordinated launch package (announcement, FAQ, internal briefing, timeline).

**What you bring:** Product brief, target audience, key messages, timeline, stakeholders.
**What Codex returns:** Launch kit with announcement draft, FAQ doc, internal briefing, stakeholder comms, timeline checklist.

---

## 6. Workflow Audits

**Use this when:** You need to audit and improve an existing workflow or process.

**What you bring:** Current process documentation, pain points, metrics, team feedback.
**What Codex returns:** Audit report with bottleneck identification, improvement recommendations, implementation plan.

---

## Key Pattern: Feed Context, Not Instructions

The recurring theme across all workflows: **provide context and ask for output**, rather than giving step-by-step instructions. Codex works best when given the raw materials (calendar, docs, messages) and asked to produce a structured artifact. The human's role shifts from creating to reviewing and editing.
