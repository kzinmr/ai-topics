# Will Brown (@willccbb) — RL-Harness Lifecycle Tweet

**Source:** https://x.com/willccbb/status/2045958417073029546
**Date:** 2026-04-30
**Author:** Will Brown (@willccbb), Research Lead at Prime Intellect

## Original Tweet

> it's not RL vs harnesses
> it's the RL-harness lifecycle
> how much would you guess the labs are now spending on "openclaw envs"?
> sadly, harness paradigms can only evolve half a model generation at a time. they need to half-work in order to get enough attention to be trained on
> the best harness ideas are those which didn't actually work yet, but which would be amazing in theory if you did the RL
> CoT, ReAct, parallel tools, claude code, compaction, subagents, RLMs
> also why i'm bearish on bolt-on memory. not a clean enough rollout loop.

## Key Claims

1. **RL vs Harness is a false dichotomy** — they co-evolve in a lifecycle
2. **Labs are investing heavily in OpenClaw-like environments** for RL training
3. **Harness paradigms evolve half a model generation at a time** — they need to half-work to get training attention
4. **Best harness ideas don't fully work yet** — but would be amazing if RL'd
5. **Bolt-on memory is problematic** — not a clean enough rollout loop for RL

## Context

Will Brown is a leading voice in open-source RL infrastructure (verifiers library, PRIME-RL, Environments Hub). His perspective comes from building the actual training environments for agentic RL, not from product-side agent frameworks.
