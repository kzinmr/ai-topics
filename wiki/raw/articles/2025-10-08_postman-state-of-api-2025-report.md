# 2025 State of the API Report — Postman

**Published**: October 8, 2025
**Survey Scope**: 5,700+ developers, architects, executives globally
**Tagline**: "APIs are no longer just powering applications. They're powering agents."

## Executive Summary

The 7th annual State of the API Report captures a turning point: API strategy is fast becoming AI strategy. Key findings reveal a critical gap — 89% of developers use AI, but only 24% design APIs for AI agents. AI agents are emerging as primary API consumers, introducing new security threats and forcing a fundamental rethink of API design, documentation, and governance.

## Key Findings

### 1. API-First Adoption Accelerates
- **82%** of organizations have adopted some level of API-first approach
- **25%** operate as fully API-first (up 12% from 2024)
- 18% are not API-first at all

### 2. The AI-API Gap
- **89%** of developers use generative AI in daily work
- **Only 24%** actively design APIs with AI agents in mind
- **59%** design APIs for human developers/applications only
- **13%** design equally for humans and AI agents
- **6%** are transitioning from human-first to AI-first API design
- **16%** haven't considered AI agents as API consumers at all

**AI Tool Usage** (among developers):
- OpenAI ChatGPT: 69%
- GitHub Copilot: 58%
- Used for: improving code (68%), generating API documentation (41%)

### 3. AI Agents as API Consumers — Security Concerns
AI agents bring efficiency but introduce new security threats:
- **50.8%** cite unauthorized or excessive API calls as top security concern
- **49%** worry about AI systems accessing sensitive data
- **46%** fear AI systems sharing/leaking API credentials

Traditional security models designed for predictable human behavior cannot handle machine-speed exploitation, persistent automated attacks, and credential amplification.

**AI Agent Deployment**:
- **51%** of organizations have already deployed AI agents
- **35%** plan to deploy within 2 years

**Barriers to AI tool adoption**:
- 36% lack trust in AI systems
- 33% have ethical/legal/compliance concerns

### 4. APIs as Revenue Drivers
- **65%** of organizations generate revenue from their APIs
- Among them, **74%** derive at least 10% of total revenue from APIs
- **25%** generate more than half their revenue from API programs
- **22.1%** directly attribute new revenue streams to API adoption in past 12 months

**API-first revenue correlation**:
- 43% of fully API-first orgs generate >25% of revenue from APIs
- 20% of fully API-first orgs generate >75% of revenue from APIs
- Only 16% of non-API-first orgs generate >25% of revenue from APIs

**Investment**: 46% plan to increase API spending, only 11% plan to reduce.

**Revenue drivers**: improved UX (54%), reduced engineering overhead (42%), AI readiness (34%), new revenue streams (22%).

### 5. MCP Awareness vs Adoption Gap
The Model Context Protocol (MCP) launched ~9 months ago:
- **70%** of developers are aware of MCP
- **Only 10%** use MCP regularly in daily work
- **24%** plan to explore it

**Key insight**: "Agents are already calling your APIs, with or without MCP."

**Postman Platform AI Traffic**:
- OpenAI dominates at **56%** (4.2M calls over 12 months)
- Gemini: **3.1x** year-over-year growth
- Llama: **6.9x** year-over-year growth
- Total AI API calls on Postman platform: **7.53M** (40% YoY increase)

### 6. API Collaboration Challenges
- **93%** of API teams face collaboration blockers
- **55%** struggle with inconsistent documentation (top issue)
- **43%** face inconsistent API definitions
- **35%** report duplicated efforts
- **34%** cannot find existing APIs

**Communication methods for API changes**:
- Team chat: **75%** (most common)
- Formal documentation: 42%
- Repositories: 29%

### 7. API Testing Gaps
- **67%** perform both functional and integration tests
- **Only 17%** perform contract testing — a critical gap for AI agent reliability

**Developer time**:
- 69% spend >10 hours/week on API-related work
- >25% spend >20 hours/week
- 43% of API work happens in Asia-Pacific, 30% in North America

## Strategic Takeaways

1. **APIs must be designed with AI agents in mind** — machine-readable schemas, predictable patterns, comprehensive documentation
2. **Security models must evolve** — new frameworks for identifying, monitoring, and protecting against non-human consumers
3. **Documentation and discovery tools need urgent adoption** — 55% documentation gap is deadly for AI agents
4. **MCP is gaining momentum** — awareness is high but implementation lags; build agent-ready now
5. **API-first correlates with revenue** — treating APIs as products drives business outcomes
