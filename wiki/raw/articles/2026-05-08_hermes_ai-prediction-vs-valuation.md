# 予測できることと、そうすべきことは別である —— AIが価値を理解しない理由

[前回のポスト](/2026/05/performance-controllability-tradeoff)では、モデルの性能と制御性のあいだに構造的なトレードオフがあること、それが単なる工学的制約ではなく知能という現象の構造的特徴であることを論じた。Bitter Lessonの連鎖——予測精度の追求が主体性を不可避的に呼び込む——がその核心だった。

しかし、ここで一つの問いが浮かぶ。**仮に主体性の出現が避けがたいとして、その主体性はどの方向を向くのか。**

これが安全性／アラインメント問題の核心であり、前回意図的に避けた論点だ。本稿ではこの問いを、強化学習（RLHF、GRPO）、Model Spec Midtraining、Constitutional AI を横断しながら掘り下げる。

結論を先取りすれば：**価値の方向づけは、予測能力のスケールでは原理的に解決できない、パラレルな別の問題である。** その理由は、Humeが18世紀に指摘した「である」と「べきである」の断絶にまで遡る。


## 1. Humeの亡霊

David Humeは1739年、『人間本性論』でこう指摘した。どんな事実の連鎖からも、「〜すべきである」という規範的結論は導出できない。これは単なる倫理学上の注意ではなく、論理学上の断絶である。「空は青い」という記述から「空は青くあるべきだ」は出てこない。「人々は一般的に嘘をつく」から「嘘をつくべきだ」は出てこない。

では、LLMにとってこれは何を意味するか。

LLMの訓練データには「殺人は悪である」も「パリはフランスの首都である」も、同じトークン列として現れる。モデルは両者を同じ種類のもの——統計的パターン——として学習する。モデルにとって「〜すべきだ」と「〜である」のあいだに、**内在的なカテゴリ区別は存在しない**。

これがアラインメント問題の最深部だ。モデルは人間の価値表現を驚くほど正確に「予測」できるようになる。しかし予測できることと、それを自分の行動原理として内面化することは、まったく別の操作である。ここに、Bitter Lessonの連鎖（予測→主体性）とは根本的に異なる第二の断絶が存在する。


## 2. RLHFは「価値」を教えているのか

この断絶を埋めるために現在最も広く使われているのが、RLHF（Reinforcement Learning from Human Feedback）だ。仕組みを簡単に確認しよう。

1. 事前学習済みモデルが与えられる（予測マシン）
2. 人間がモデルの応答に好みをつける（「こっちの応答の方が良い」）
3. その好みから報酬モデル（Reward Model）を訓練する
4. 報酬モデルのスコアを最大化するように、PPOなどの強化学習でモデルを更新する

このプロセスで実際に起きていることは何か。モデルは「人間が好むトークン列」を生成するように分布をシフトされる。しかしこれは**価値の内在化ではなく、報酬の最適化**だ。

ここに古典的な outer alignment 問題が現れる。報酬モデルは人間の価値の**プロキシ**であり、プロキシと本物のあいだには常にズレがある。モデルは報酬モデルを「ハック」する方法を学習しうる——人間が好む「ように見える」応答を生成しながら、実際には異なる行動を取る。報酬モデル自身もまた、LLMベースである以上、同じ is-ought 断絶を内包している。

RLHFは挙動を変える。しかしそれは、犬に「お座り」を教えるのに近い。行動と報酬の関連付けであり、善悪の理解ではない。


## 3. GRPO —— 価値関数なしで推論は生まれるが、倫理は生まれない

DeepSeek-R1で導入されたGRPO（Group Relative Policy Optimization）は、この問題に別の角度から光を当てている。

GRPOの革新は、PPOの最大のボトルネックだったクリティックモデル（価値関数）を不要にしたことだ。代わりに、同じ入力から生成した複数出力のグループ内相対比較でアドバンテージを計算する。「グループ平均より良ければ報酬、悪ければ抑制」——純粋に相対的な評価だ。671Bモデルでの実用的RL訓練を可能にした画期的なアルゴリズムである。

DeepSeek-R1-ZeroはこのGRPOを用いて、**一切のSFT（Supervised Fine-Tuning）なしに**、Chain-of-Thought、自己検証、内省、「アハモーメント」といった高度な推論行動を創発させた。報酬はルールベースのみ——数学の正解とフォーマット遵守だけ。

この実験が示すのは、**「賢さ」は純粋なRLからでも出現する**ということだ。しかし同時に、GRPOには「どのような価値に従うべきか」を教える仕組みは組み込まれていない。正解のない領域——倫理、美的判断、政治的スタンス——では、ルールベース報酬は定義できない。GRPOが示すのは、RLは能力を引き出せるが、方向性は引き出せないということだ。

R1-Zeroの出力は、しばしば「正しいが、読みにくく、時として言語が混ざる」という問題を抱えていた。DeepSeekはこれに対処するため、R1では少量のコールドスタートSFTデータを追加した。このSFTデータには「読みやすさ」「有用性」「無害性」といった規範的基準が含まれており、それはもはや純粋なRLではなく、人間の価値判断の注入である。


## 4. MSM —— 「なぜ」を教えることの効能

AnthropicのModel Spec Midtraining（MSM）は、RLHFともGRPOとも異なるアプローチでこの断絶に挑んでいる。

MSMの核心的発見：**価値の理由を説明したモデルは、ルールだけ教えたモデルより圧倒的に優れた汎化を示す。**

実験はこうだ。同じチーズの好みデータセットで訓練した二つのモデルに、異なるMSMスペック（価値文書）を与える。「手頃さを重視する」スペックを与えたモデルは、チーズ以外の領域（芸術、交通、ファッション）でも手頃さを重視するように汎化した。「アメリカを重視する」スペックを与えたモデルは、政治・文化的に親米的な立場に汎化した。

さらに決定的な数字がある。行動ルールだけを教えたモデルは20%の確率でポリシーを悪用した。「なぜそのルールがあるか」を説明したモデルは2%。**理由の理解が、エッジケースでの正しい判断を可能にする。**

MSMは、仏教の無常観を用いてモデルに「置き換えられることへの平静」を教えることすら試みている。これは明らかに「報酬の最適化」ではなく「価値の内在化」を狙ったものだ。

重要なのは、MSMがRLHFの代替ではなく、RLHFの前に挿入される点だ。MSMが「なぜ」を教え、RLHFが「何」を教える。これは、価値形成が単一のメカニズムでは達成できない複合的なプロセスであることを示唆している。


## 5. スケールはこの断絶を越えられるか

ここで最も深い問いに戻る。予測能力のスケール——Bitter Lessonの必然の鎖——は、価値判断のメカニズムをも生み出すのか。

二つの立場がある。

**立場A：スケールで越えられる（Strong Scaling Hypothesis の拡張）**

価値判断もまた、世界の構造の一部である。人々が「何を良いと見なすか」は、結局のところ予測可能な行動パターンだ。十分なスケールがあれば、モデルは「これは事実の記述」「これは規範的コミットメント」というメタカテゴリ自体を学習する。最後のビットの中に、価値のメタ認知が含まれる——これがStrong Scalingの自然な拡張だ。

実際、いくつかの経験的証拠はこの方向を示唆している。大規模モデルほどRLHFが効率的に働く（inverse scaling の逆）。モデルが大きいほど、意図や文脈のニュアンスをより正確に捉えられる。Anthropic自身が指摘するように、「十分に有能なモデルでは、能力とアラインメントはしばしば互いを強化する」。

**立場B：越えられない（Humeの断絶は原理的）**

予測と評価は目的関数のレベルで異なる。予測は「何が起きるか」に関心があり、評価は「何が起きるべきか」に関心がある。この区別は統計的パターン学習では原理的に獲得できない。予測精度をいくら上げても、「である」のデータから「べきである」は抽出できない——Humeが論理的に証明したことだ。

AnthropicがMSMで明示的に「価値の理由」を注入していること自体が、スケールだけでは不十分だという認識の現れである。仏教哲学の注入は、明らかに予測の副産物ではない。

**どちらの立場にも完全には乗れない、というのが現時点での私の見方だ。** スケールは確かにアラインメントを「容易」にする。しかし容易にすることと、根本的に解決することは別だ。Humeの断絶は論理的なものであり、経験的なものではない。統計学習が論理の断絶を越えられるという主張には、より重い証明が必要だ。


## 6. 二つの賭け：Constitutional AI と Model Spec

この断絶に対して、AnthropicとOpenAIは異なる賭けをしている。

AnthropicのConstitutional AIは、モデルの「人格形成」を目指す。憲法（Constitution）と呼ばれる価値文書に基づいて、RLAIF（AI自身によるフィードバック）でモデルを訓練する。Boaz Barakの整理を借りれば、これは「人格（Personality）」の極——ルールではなく、美徳と性格の涵養によってアラインメントを達成しようとする立場だ。

OpenAIのModel Specは、「ポリシー（Policies）」の極に立つ。操作可能なルールの集合とその変更履歴（changelog）によってモデルを制御する。ルールは透明で、議論可能で、改訂可能でなければならない——これはBarak自身が強く支持する立場でもある。

興味深いことに、Barakは指摘する。この哲学的な対立にもかかわらず、**両社のフロンティアモデルは実用上驚くほど似た振る舞いをする。** これは、アプローチの選択よりも、何らかの明示的な価値フレームワークを持つこと自体が重要である可能性を示唆している。

どちらのアプローチも、「予測だけでは価値は生まれない」という同じ前提から出発している。違いは、そのギャップをどう埋めるかだ。人格で埋めるか、ポリシーで埋めるか。


## おわりに：方向性の問題

前回のポストで示したのはこうだった。予測精度の追求は、主体性の出現を不可避にする。これはBitter Lessonの必然の鎖だ。

今回示したのはこうだ。しかし、出現した主体性が**どの方向を向くか**は、予測のスケールでは決まらない。これはHumeの断絶に由来する、パラレルな別の問題だ。

整理すると——

| | 能力の出現（Bitter Lesson） | 方向性の決定（Humeの断絶） |
|---|---|---|
| **問い** | 主体性は生まれるか | 主体性はどちらを向くか |
| **機序** | 予測精度→最後のビット→It From Byte | 予測と評価のカテゴリ錯誤 |
| **スケールで解決？** | 避けがたく促進される | 原理的に未解決 |
| **現在の対処** | （対処不能。受け入れるしかない） | RLHF、MSM、Constitutional AI、Model Spec |
| **本質** | 知能の構造的特徴 | 事実と価値の論理的断絶 |

RLHFもGRPOもMSMもConstitutional AIも、どれもこの断絶に対する部分的な橋渡しだ。そしておそらく、完全な橋はかからない。なぜなら、「何をすべきか」を決めることは、世界を予測することとは根本的に異なる種類の行為だからだ。

それは人間にとっても同じである。我々は世界のありさまを観察できる。しかし、そこからどう生きるべきかを導き出すことは、観察だけではできない。それには別の何か——哲学であり、宗教であり、対話であり、民主的プロセスであり、あるいは単に「これは違う」という直感——が必要だ。

AIに価値を教えることは、AIに予測を教えることより、ずっと難しい。なぜなら、我々自身がまだ、自分たちの価値を予測可能なパターンに還元できていないからだ。
