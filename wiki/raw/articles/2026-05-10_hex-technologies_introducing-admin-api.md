Today, we’re excited to introduce a new set of APIs for Hex Admins — a powerful way to programmatically manage your workspace, streamline governance, and integrate more deeply with your existing data platform.
Every growing organization runs into the same challenge: admin tasks that scale linearly with team size. What worked for 50 users becomes untenable at 500 or 5,000. Manual permission updates, group changes, and project clean-up don’t scale, leading to significant overhead.
Our new Admin REST APIs help teams cut through the clutter
with automation, auditability, and scale
. You can now programmatically manage users, groups, projects, and more — giving you time back to focus on building impactful data products.
knowledge
Admin REST APIs are available on the
Team and Enterprise plans
. The advanced Observability API is available to Enterprise customers only.
Contact Sales
to learn more.
Whether you’re wrangling dozens of users or hundreds of projects, the Admin APIs make it easy to:
Automate user and group management (no directory sync required)
Enforce consistent access policies across teams
Integrate Hex into your internal IT or platform tooling
Manage workspace collections at scale
Rotate credentials for data connections
Key API capabilities
With Hex’s REST APIs, you can replace error-prone manual workflows with fast, automated processes that scale effortlessly:
👥
Users & Groups:
Add, remove, and manage group memberships to streamline onboarding and offboarding, and keep access aligned with your org chart.
🗂️
Collections:
Create and update project collections with specific sharing settings to enforce workspace structure and permission boundaries.
🔐
Data Connections:
Rotate credentials, update configurations, and manage access to secure data sources — no UI clicks required.
📊
Project Usage Insights
(Enterprise plan only):
Use the new
queriedTables
endpoint to understand which data tables a project queries. This comes in handy for audits, lineage mapping, and impact analysis.
Ready to transform your admin workflows? Check out our
API overview
and
API reference
docs.

We're exploring the possibility of offering a
Model Context Protocol (MCP) server
to provide a hosted orchestration layer for managing users, permissions, and policies at scale — backed by Hex’s Admin APIs.
