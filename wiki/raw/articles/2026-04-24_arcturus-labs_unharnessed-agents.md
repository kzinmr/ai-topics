# Unharnessed Agents Power the Future of AI Products

John Berryman, Founder of Arcturus Labs. April 24, 2026. 7 min read.

## Agents and Agent Harnesses

In 2025 we wrestled with the notion of an agent. Was it a workflow? Was it a `while` loop? Workflows were too rigid — the agent couldn't follow its own logic. The `while` loop agent was too free — the model would wander off.

By end of 2025, models improved enough that `while` loop agents won. Agent harnesses popped up around code development — human-in-the-loop, batteries-included `while` loops with opinions on context management, filesystem/OS integration, sandboxing, skills, and MCP. Examples: Cursor agent, Claude Code, Codex, OpenCode. OpenClaw showed how much control could be ceded to the agent.

## The Harness Is the Wrong Frame

Three problems:

1. **Opaque and opinionated** — Every provider's harness feels the same but differs in context compaction/pruning, memory management. You rarely control or understand what it's doing.

2. **Misleading name** — "Agent harness" implies the agent lives inside a harness. But the agent IS the while loop, LLM calls, tool calls, context management, skills, MCP integrations. The agent is the WHOLE COMPOSITION. The agent IS the "agent harness."

3. **Too narrow** — Fixation on "agent harness = coding tool in IDE/terminal" stifles product possibilities. The same machinery can work on any task addressable via API.

## Agents Should Leave the IDE

The agent should follow you everywhere — write code, draft blog posts, sort email, voice-navigate your phone, join meetings and take notes, perform background lookups, organize Obsidian notes. The agent should auto-generate UIs for any data/service. In this future, "the internet for humans goes away" — we get locally-generated interfaces while agents navigate the internet on our behalf.

## Skills Are the New Programs

Skills are the new programs. English is the new programming language. The agent is the new runtime.

- **Skills can create other skills** — Berryman uses a `create-skill` skill to capture workflows
- **Skills can improve existing skills** — An `improve-skill` skill analyzes traces to update other skills
- **Skills can retrieve and compose other skills on the fly** — A `build-anything` skill that downloads missing skills and incorporates them

Non-technical users can express intent in plain language, and the agent translates it into a skill — "software development for the everyman."

## How We Get There

1. **Everyone builds their own agent** — Develop intuition for context management, memory, skill writing, agentic operation. Parallelize learning across use cases. Your agent will be transparent and extensible to any environment/modality/complexity.

2. **Work toward an agentic standard** — Define standard modules (compaction, pruning, memory, auth). Standardize how agents connect to environments — "any environment, any modality, any complexity."

Once standardized, we won't build agents from scratch — we'll assemble exactly the agent we need from a small set of well-understood primitives, "programming" it with skills.

## Let's Unharness This Together

Form a standards committee. Catalog real agent behaviors across tasks, environments, and users. Outline a new standard for how agents are built and expected to behave. Unlock the next level: not just more powerful agents, but entire "agencies" — business processes run by coordinated swarms of agents working alongside humans.
