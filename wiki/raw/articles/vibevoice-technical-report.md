# VibeVoice Technical Report

VibeVoice is a novel framework from Microsoft Research for generating expressive, long-form, multi-speaker conversational audio. The technical report was accepted as an **ICLR 2026 Oral** presentation.

## Core Innovation: Next-Token Diffusion

VibeVoice adapts the next-token diffusion framework (first introduced in LatentLM, Sun et al. 2024) for speech synthesis. This treats continuous data (audio latents) as sequences of vectors generated autoregressively through diffusion processes.

## Key Technical Details

- **Continuous Speech Tokenizer**: Operates at an ultra-low 7.5 Hz frame rate
- **Compression ratio**: 3200x downsampling from 24kHz input — 80x better than Encodec
- **Context window**: 65,536 tokens (trained with curriculum: 4K → 16K → 32K → 64K)
- **Maximum output**: ~90 minutes of speech with up to 4 speakers
- **Architecture**: Qwen2.5 LLM (1.5B base) + Dual tokenizers (acoustic & semantic) + Diffusion head (4 layers, ~123M parameters)

## Dual Tokenizer Design

1. **Acoustic Tokenizer**: Based on σ-VAE variant (from LatentLM), mirror-symmetric encoder-decoder with 7 stages of modified Transformer blocks. ~340M parameters each for encoder/decoder.
2. **Semantic Tokenizer**: Mirrors acoustic architecture without VAE components. Trained with ASR proxy task.

## Model Variants

| Model | Base LLM | Speakers | Max Duration | Languages |
|-------|----------|----------|-------------|-----------|
| VibeVoice-1.5B (Full) | Qwen2.5-1.5B | Up to 4 | ~90 min | English, Chinese |
| VibeVoice-7B (Reported) | Qwen2.5-7B | Up to 4 | ~90 min | English, Chinese |
| VibeVoice-Realtime-0.5B | Qwen2.5-0.5B | Single | Streaming | English |
| VibeVoice-Large | — | — | ~45 min | Disabled |

## Audio Quality

- **ICLR 2026 Oral** acceptance
- Outperforms commercial systems (Gemini 2.5 Pro TTS, ElevenLabs V3) in subjective evaluations
- WER: 1.29% (VibeVoice-7B) vs 1.73% (Gemini)
- Generated speech captures breaths, lip smacks, and spontaneous expressions
- Realtime variant achieves ~160-300ms time-to-first-audio

## Data Pipeline

- Developed an annotation pipeline generating pseudo transcriptions and turn-taking labels for extensive podcast data
- Enables learning authentic conversational dynamics (natural turn-taking, pacing, non-lexical cues)

## Safety Features

- Audible disclaimer: "This segment was generated by AI" embedded in every file
- Imperceptible watermarking for provenance verification
- Inference request hashing and logging for abuse detection
- Repository disabled (as of Sep 2025) due to misuse concerns — Microsoft disabled the GitHub repo after discovering uses inconsistent with stated intent

## Limitations

- No overlapping speech modeling (speakers don't talk over one another)
- May inherit biases from Qwen2.5 base model
- Research-only recommendation (not for commercial use without further testing)
- Primarily English/Chinese; other languages may produce unpredictable results
