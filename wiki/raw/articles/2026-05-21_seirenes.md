# Seirênes: Adversarial Self-Play with Evolving Distractions for LLM Reasoning

**Source:** arXiv:2605.11636v1 (May 21, 2026) | [Paper](https://arxiv.org/html/2605.11636v1) | [Code: MiliLab/Seirenes](https://github.com/MiliLab/Seirenes)

**Authors:** Chi Zhang, Haibo Qiu, Qiming Zhang, Yufei Xu, Xinbo Gao, Jing Zhang

## Overview

Seirênes is a self-play RL framework that transforms contextual interference (misleading hints, distracting context) from a failure mode into an internal training signal for more robust reasoning. A single shared-parameter LLM plays two competing roles — Adversary and Reasoner — in a co-evolutionary loop.

## Core Mechanism

- **Adversary**: Generates plausible but misleading hints designed to derail reasoning
- **Reasoner**: Must solve problems under perturbed context, recovering core logic despite interference
- **Co-evolution**: Adversary rewarded for reducing Reasoner's success; Reasoner rewarded for solving correctly

Named after the Sirens of Greek mythology, whose alluring songs misled sailors — here, misleading hints tempt the Reasoner to follow plausible but wrong paths.

## Method

1. **R1 (Clean reasoning)**: Reasoner solves original question → reward
2. **R2 (Adversarial hints)**: Adversary generates G₂ misleading hints → each appended to question
3. **R3 (Hint-conditioned reasoning)**: Reasoner solves under adversarial context → reward

Uses GRPO as base RL algorithm. Three objective streams: Clean Reasoner, Adversary (negative GRPO), Robustness Reasoner.

## Key Design Elements

- **Separate update streams**: Gradients kept separate via per-stream FIFO buffers, preserving credit assignment
- **Mastery-aware sampling**: Questions too easy (all rollouts correct) moved to "mastered set" and deprioritized
- **Latency-aware scheduling**: R1 and R2 concurrent; R3 depends only on R2 outputs

## Results

- **Backbones**: Qwen2.5-7B-Instruct, Qwen3-4B-Instruct, Qwen3-30B-A3B-Instruct
- **Training data**: DAPO-Math-17K and OpenR1-Math
- Seirênes-trained models achieve stronger robustness against adversarial context without sacrificing clean accuracy

## Significance

Addresses a key limitation of RLVR-trained reasoning models: fragility under non-idealized contexts. Structural reformulations reduce accuracy 27-31%; irrelevant retrievals can drop accuracy from 96% to 65%. Seirênes uses these vulnerabilities as an online curriculum rather than just testing robustness.
